### Title
`max_nonce_for_validation_skip` config field is silently ignored by `skip_stateful_validations`, allowing `__validate__` bypass for nonce=1 invoke transactions even when the skip is operator-disabled — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

`StatefulTransactionValidatorConfig` exposes a `max_nonce_for_validation_skip` field that is supposed to bound (or disable) the deploy-account+invoke UX skip of `__validate__`. The Apollo gateway's `skip_stateful_validations` free function never reads this field; it hardcodes `Nonce(Felt::ONE)`. An operator who sets `max_nonce_for_validation_skip = 0` to enforce strict validation for every transaction will find the gateway still skips `__validate__` for any nonce-1 invoke from an undeployed account, admitting the transaction to the mempool without signature verification.

### Finding Description

`StatefulTransactionValidatorConfig` declares:

```rust
pub max_nonce_for_validation_skip: Nonce,
```

with default `Nonce(Felt::ONE)`, serialised into the config schema and deployment configs. [1](#0-0) 

The native-blockifier path (`PyValidator::should_run_stateful_validations`) correctly gates the skip on this value:

```rust
let nonce_small_enough_to_qualify_for_validation_skip =
    tx_nonce <= self.max_nonce_for_validation_skip;
let skip_validate = deploy_account_not_processed
    && is_post_deploy_nonce
    && nonce_small_enough_to_qualify_for_validation_skip;
``` [2](#0-1) 

The Apollo gateway path calls `skip_stateful_validations` as a **free function** — it does not receive `self` and therefore cannot access `self.config.max_nonce_for_validation_skip`. Instead it hardcodes the threshold:

```rust
if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
    // always skips __validate__ for nonce-1 invoke
``` [3](#0-2) 

The call site in `run_pre_validation_checks` passes no config parameter:

```rust
let skip_validate =
    skip_stateful_validations(executable_tx, account_nonce, mempool_client.clone()).await?;
``` [4](#0-3) 

A grep for `max_nonce_for_validation_skip` inside `crates/apollo_gateway/` returns **zero matches**, confirming the field is dead code in the production gateway path. [5](#0-4) 

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

When `max_nonce_for_validation_skip` is set to `0` (operator intent: disable the skip entirely), the gateway still skips `__validate__` for any invoke transaction whose nonce is 1 and whose sender has nonce 0 on-chain but a deploy-account in the mempool. The transaction is admitted without the account's `__validate__` entry point being called, meaning the account's own signature check is bypassed at the gateway layer. The transaction enters the mempool and proceeds toward execution without having been signature-verified.

### Likelihood Explanation

The default value of `max_nonce_for_validation_skip` is `Nonce(Felt::ONE)`, which coincidentally matches the hardcoded constant, so the bug is invisible in default deployments. The vulnerability surfaces only when an operator explicitly sets the field to `0` to harden the gateway — a reasonable operational action given the field is documented and exposed in the config schema. [6](#0-5) 

### Recommendation

Convert `skip_stateful_validations` from a free function into a method on `StatefulTransactionValidator` (or pass `self.config.max_nonce_for_validation_skip` as an explicit argument) so the configured threshold is actually enforced:

```rust
// inside run_pre_validation_checks
let skip_validate = self
    .skip_stateful_validations(executable_tx, account_nonce, mempool_client.clone())
    .await?;

// method body mirrors PyValidator::should_run_stateful_validations
async fn skip_stateful_validations(...) -> ... {
    if let ExecutableTransaction::Invoke(...) = tx {
        if tx.nonce() > Nonce(Felt::ZERO)
            && tx.nonce() <= self.config.max_nonce_for_validation_skip
            && account_nonce == Nonce(Felt::ZERO)
        {
            return mempool_client.account_tx_in_pool_or_recent_block(...).await...;
        }
    }
    Ok(false)
}
```

### Proof of Concept

1. Deploy the sequencer with `gateway_config.static_config.stateful_tx_validator_config.max_nonce_for_validation_skip = 0x0` (operator disables the UX skip).
2. Attacker submits a valid `DeployAccount` transaction for address `A` — this passes full validation and enters the mempool.
3. Attacker submits an `Invoke` transaction from `A` with `nonce = 1` and a **garbage signature**.
4. `skip_stateful_validations` evaluates `tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO)` → `true`; `account_tx_in_pool_or_recent_block(A)` → `true` (step 2 is in the mempool).
5. `run_validate_entry_point` is called with `execution_flags.validate = false` — `__validate__` is never executed.
6. The malformed invoke transaction is admitted to the mempool without any signature check, contradicting the operator's explicit configuration. [7](#0-6) [8](#0-7)

```rust
async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
```
