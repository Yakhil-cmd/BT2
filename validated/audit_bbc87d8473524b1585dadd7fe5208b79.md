### Title
Missing Access Control on `update_rewards` Enables Permanent Freezing of Consensus Rewards — (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the Staking contract is intended to be callable only by the Starkware sequencer (per the protocol spec), but the implementation enforces no such restriction. Any unprivileged address can call it with `disable_rewards: true`, consuming the global `last_reward_block` slot for that block and permanently preventing the sequencer from distributing consensus rewards to any staker.

---

### Finding Description

The protocol specification explicitly states:

> **access control**: Only starkware sequencer.

However, the implementation of `update_rewards` only calls `general_prerequisites()`, which checks that the contract is unpaused and the caller is non-zero — no sequencer identity check is performed. [1](#0-0)

Because `last_reward_block` is global (not per-staker), a single call to `update_rewards` for **any** staker in block N prevents any further call to `update_rewards` for **any** staker in block N. The check is: [3](#0-2) 

The spec confirms this is the intended single-call-per-block design: [6](#0-5) 

---

### Impact Explanation

An attacker calling `update_rewards(any_active_staker, disable_rewards: true)` in every block:

1. Sets `last_reward_block` to the current block number.
2. Returns early without distributing any rewards (the `disable_rewards` branch).
3. Causes the sequencer's subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`. [7](#0-6) 

No consensus rewards are ever distributed to any staker or delegator. The `unclaimed_rewards_own` field of every staker remains at zero indefinitely, and pool contracts never receive reward transfers via `update_pool_rewards`. [8](#0-7) 

**Impact**: **Permanent freezing of unclaimed yield** for all stakers and all pool delegators — a High-severity impact under the allowed scope.

---

### Likelihood Explanation

- The function is publicly callable by any non-zero address with no role or identity check.
- The attacker only needs to submit one cheap transaction per block (Starknet gas costs are low).
- A competing staker or any griefing actor can sustain this indefinitely.
- No privileged access, leaked key, or external dependency is required.

Likelihood: **High**.

---

### Recommendation

Introduce an authorized caller check in `update_rewards`. Store the sequencer address (or a designated rewards manager role) in contract storage and assert `get_caller_address() == sequencer_address` at the top of the function, consistent with the protocol specification.

---

### Proof of Concept

```
Block N:
  Attacker tx:   update_rewards(staker=0x1, disable_rewards=true)
                 → last_reward_block = N, no rewards distributed

  Sequencer tx:  update_rewards(staker=0x2, disable_rewards=false)
                 → PANICS: "Rewards already updated for this block"

Block N+1:
  Attacker tx:   update_rewards(staker=0x1, disable_rewards=true)
                 → last_reward_block = N+1, no rewards distributed

  Sequencer tx:  update_rewards(staker=0x2, disable_rewards=false)
                 → PANICS: "Rewards already updated for this block"

... repeated every block → zero consensus rewards ever distributed
```

The attacker pays only gas per block. All stakers' `unclaimed_rewards_own` and all pool reward balances remain permanently at zero for the consensus rewards phase. [9](#0-8)

    #[abi(embed_v0)]
    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
    }
