### Title
Staker Can Unilaterally Spike Commission via `set_commission_commitment` + `set_commission` to Steal Delegator Yield - (File: src/staking/staking.cairo)

### Summary
A staker can call `set_commission_commitment` to establish a commitment with `max_commission = 10000` (100%), then immediately call `set_commission` to raise commission to 100% in the same block. Because the commission value is read live from storage at reward-distribution time with no epoch delay, the very next `update_rewards` call distributes 100% of pool rewards to the staker, leaving delegators with zero yield.

### Finding Description
The `set_commission_commitment` function stores a `CommissionCommitment { max_commission, expiration_epoch }` with no restriction on how high `max_commission` can be relative to the current commission — only that `current_commission <= max_commission`. [1](#0-0) 

Once a commitment is active, `update_commission` permits any new commission value that satisfies `commission <= commitment.max_commission && commission != old_commission`, explicitly allowing increases: [2](#0-1) 

The developers themselves flag this in a code comment: [3](#0-2) 

At reward-distribution time, `calculate_staker_pools_rewards` reads the commission directly from live storage — there is no epoch-based snapshot or delay: [4](#0-3) 

The `split_rewards_with_commission` utility then deducts `commission_rewards` from the pool's share and credits it to the staker: [5](#0-4) 

**Attack sequence (two transactions, same block):**
1. Staker is operating with `commission = 0` and has attracted delegators.
2. Staker calls `set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1)`.
3. In the same block, staker calls `set_commission(commission: 10000)`.
4. The commitment is active (`current_epoch < expiration_epoch`), so the increase passes all checks and `commission` is written to storage immediately.
5. The sequencer's next `update_rewards` call reads `commission = 10000`, computes `commission_rewards = 100% × pool_rewards`, and transfers the entire pool reward to the staker.
6. Delegators receive zero yield for that distribution period.

This is structurally identical to the reported escrow bug: one party (staker ↔ buyer) unilaterally sets a fee variable (`commission` ↔ `arbitratorFee`) that immediately governs the next fund-distribution call (`update_rewards` ↔ `release`), with no consent or delay protecting the other party (delegators ↔ seller).

### Impact Explanation
Every delegator in the pool loses 100% of their accrued yield for the affected distribution period. With large pools this is a direct, material theft of unclaimed yield. Delegators cannot exit before the next `update_rewards` call because the exit flow requires `exit_intent` → wait-window → `exit_action`, all of which span multiple epochs.

**Impact: High — Theft of unclaimed yield.**

### Likelihood Explanation
Any registered staker who has opened a delegation pool can execute this in two sequential transactions. No privileged role, leaked key, or external dependency is required. A rational attacker would first build a large delegated pool (advertising 0% commission), then execute the spike just before a reward block to maximise stolen yield. The minimum commitment window is one epoch, which is sufficient.

### Recommendation
Commission increases should not take effect in the same epoch they are set. Specifically:

- Store the pending increased commission in a separate `pending_commission` field (analogous to the `proposedFee` recommendation in the external report).
- Only promote `pending_commission → commission` at the start of the next epoch (or after a mandatory delay epoch), so delegators have time to observe the change and exit if desired.
- Alternatively, prohibit commission increases entirely (only allow decreases), even when a commitment is active, and restrict the commitment to serve only as a lower-bound guarantee.

### Proof of Concept
```
// 1. Staker stakes with 0% commission and opens a pool.
staking.stake(amount, pool_enabled: true, commission: 0);
// Delegators join and accrue rewards over several epochs.

// 2. Staker sets a commitment allowing up to 100%.
staking.set_commission_commitment(max_commission: 10000, expiration_epoch: current_epoch + 1);
// Passes: current_commission (0) <= max_commission (10000). ✓

// 3. Staker immediately raises commission to 100% in the same block.
staking.set_commission(commission: 10000);
// Passes: commitment active, 10000 <= 10000, 10000 != 0. ✓
// commission written to storage immediately.

// 4. Sequencer calls update_rewards in the next block.
//    calculate_staker_pools_rewards reads commission = 10000.
//    split_rewards_with_commission → commission_rewards = pool_rewards * 10000/10000 = pool_rewards.
//    pool_rewards sent to pool = 0.
//    Delegators receive nothing; staker receives 100% of pool yield.

        fn set_commission(ref self: ContractState, commission: Commission) {
            // Prerequisites and asserts.
            self.general_prerequisites();
            assert!(commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);

            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            if let Option::Some(old_commission) = staker_pool_info.commission.read() {
                self
                    .update_commission(
                        :staker_address, :staker_pool_info, :old_commission, :commission,
                    );
            } else {
                staker_pool_info.commission.write(Option::Some(commission));
                self.emit(Events::CommissionInitialized { staker_address, commission });
            }
        }

        fn set_commission_commitment(
            ref self: ContractState, max_commission: Commission, expiration_epoch: Epoch,
        ) {
            self.general_prerequisites();
            assert!(max_commission <= COMMISSION_DENOMINATOR, "{}", Error::COMMISSION_OUT_OF_RANGE);
            let staker_address = get_caller_address();
            let staker_info = self.internal_staker_info(:staker_address);
            assert!(staker_info.unstake_time.is_none(), "{}", Error::UNSTAKE_IN_PROGRESS);
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            assert!(staker_pool_info.has_pool(), "{}", Error::MISSING_POOL_CONTRACT);
            let current_epoch = self.get_current_epoch();
            if let Option::Some(commission_commitment) = staker_pool_info
                .commission_commitment
                .read() {
                assert!(
                    !self.is_commission_commitment_active(:commission_commitment),
                    "{}",
                    Error::COMMISSION_COMMITMENT_EXISTS,
                );
            }
            // Staker must have a commission since it has a pool.
            let current_commission = staker_pool_info.commission();
            assert!(current_commission <= max_commission, "{}", Error::MAX_COMMISSION_TOO_LOW);
            assert!(expiration_epoch > current_epoch, "{}", Error::EXPIRATION_EPOCH_TOO_EARLY);
            assert!(
                expiration_epoch - current_epoch <= self.get_epoch_info().epochs_in_year(),
                "{}",
                Error::EXPIRATION_EPOCH_TOO_FAR,
            );
            let commission_commitment = CommissionCommitment { max_commission, expiration_epoch };
            staker_pool_info.commission_commitment.write(Option::Some(commission_commitment));
            self
                .emit(
                    Events::CommissionCommitmentSet {
                        staker_address, max_commission, expiration_epoch,
                    },
                );

pub(crate) fn split_rewards_with_commission(
    rewards_including_commission: Amount, commission: Commission,
) -> (Amount, Amount) {
    let commission_rewards = compute_commission_amount_rounded_down(
        :rewards_including_commission, :commission,
    );
    let pool_rewards = rewards_including_commission - commission_rewards;
    (commission_rewards, pool_rewards)
}
