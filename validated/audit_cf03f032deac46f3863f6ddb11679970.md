### Title
`key_manager` Role Can Bypass Owner-Controlled Pause via `start_hashchain` - (File: `engine/src/contract_methods/admin.rs`)

---

### Summary

The `start_hashchain` function is gated by `require_key_manager_only`, yet it unconditionally sets `state.is_paused = false`, unpausing the engine. Every other pause/unpause path in the contract is restricted to `require_owner_only`. This gives the lower-privileged `key_manager` role the ability to lift a security-critical pause that only the owner should control.

---

### Finding Description

The Aurora Engine has two distinct privileged roles stored in `EngineState`:

- `owner_id` — the high-trust administrator that controls upgrades, ownership transfer, pause/resume, and key-manager assignment.
- `key_manager` — a lower-trust operator whose documented purpose is managing relayer function-call access keys (`add_relayer_key`, `remove_relayer_key`). [1](#0-0) 

The pause/resume surface is consistently owner-gated:

```rust
// pause_contract
require_owner_only(&state, &env.predecessor_account_id())?;
// resume_contract
require_owner_only(&state, &env.predecessor_account_id())?;
``` [2](#0-1) 

However, `start_hashchain` is gated only by `require_key_manager_only`, and it terminates by writing `state.is_paused = false` back to storage:

```rust
pub fn start_hashchain<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    let mut state = state::get_state(&io)?;
    require_paused(&state)?;
    require_key_manager_only(&state, &env.predecessor_account_id())?;
    // ... hashchain initialization ...
    state.is_paused = false;
    state::set_state(&mut io, &state)?;
    Ok(())
}
``` [3](#0-2) 

`require_key_manager_only` enforces only that the caller equals `state.key_manager`; it has no relationship to `owner_id`: [4](#0-3) 

The `key_manager` is set exclusively by the owner via `set_key_manager`, which itself requires `require_owner_only`. The key manager's intended scope is relayer-key lifecycle management, not contract-pause control. [5](#0-4) 

---

### Impact Explanation

**High — Temporary (or worse, permanent) bypass of a security-critical pause.**

The owner's only emergency lever is `pause_contract`. If the owner pauses the engine in response to a live exploit (e.g., a bridge accounting bug, a connector vulnerability, or an ERC-20 mirror accounting flaw), the `key_manager` can immediately call `start_hashchain` to lift that pause. Any in-flight attacker can then continue draining funds or freezing balances that the pause was meant to protect. If the underlying vulnerability allows direct fund theft, the impact escalates to Critical.

---

### Likelihood Explanation

**Medium.** The `key_manager` is a trusted operator, but it is explicitly a *less-trusted* role than the owner — it is not expected to have pause authority. A compromised `key_manager` key (e.g., a hot-wallet key used for relayer operations) or a malicious operator acting within their granted scope can exploit this without any owner involvement. The design inconsistency means the attack surface exists by construction, not only under extreme compromise scenarios.

---

### Recommendation

Replace `require_key_manager_only` in `start_hashchain` with `require_owner_only`, consistent with every other function that modifies `state.is_paused`. If the hashchain-start workflow genuinely requires key-manager involvement, add a dual-check (`require_owner_only` **and** `require_key_manager_only`) or split the hashchain initialization from the unpause step so the unpause remains owner-exclusive.

---

### Proof of Concept

1. Owner calls `pause_contract` (e.g., in response to a discovered vulnerability). `state.is_paused = true`.
2. Key manager (compromised or malicious) calls `start_hashchain` with any syntactically valid `StartHashchainArgs` whose `block_height` satisfies `block_height <= env.block_height()`.
3. `require_paused` passes (contract is paused). `require_key_manager_only` passes (caller is the key manager).
4. `start_hashchain` writes `state.is_paused = false` and commits state.
5. The engine is now running again. The owner's emergency pause has been silently lifted. Any exploit the pause was meant to block can proceed. [3](#0-2) [6](#0-5)

        let key_manager =
            serde_json::from_slice::<RelayerKeyManagerArgs>(&io.read_input().to_vec())
                .map(|args| args.key_manager)
                .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;

            return Err(errors::ERR_SAME_KEY_MANAGER.into());
        }

        state.key_manager = key_manager;
        state::set_state(&mut io, &state)?;

        Ok(())
    })
}

#[named]
pub fn start_hashchain<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    let mut state = state::get_state(&io)?;
    require_paused(&state)?;
    require_key_manager_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let args = StartHashchainArgs::try_from_slice(&input).map_err(|_| errors::ERR_SERIALIZE)?;
    let block_height = env.block_height();

    // Starting hashchain must be for an earlier block
    if block_height < args.block_height {
        return Err(errors::ERR_ARGS.into());
    }

    let mut hashchain = Hashchain::new(
        state.chain_id,
        env.current_account_id(),
        args.block_height + 1,
        args.block_hashchain,
    );

    if hashchain.get_current_block_height() < block_height {
        hashchain.move_to_block(block_height)?;
    }

    hashchain.add_block_tx(
        block_height,
        function_name!(),
        &input,
        &[],
        &Bloom::default(),
    )?;
    crate::hashchain::save_hashchain(&mut io, &hashchain)?;

    state.is_paused = false;
    state::set_state(&mut io, &state)?;

    Ok(())
}

fn require_key_manager_only(
    state: &state::EngineState,
    predecessor_account_id: &AccountId,
) -> Result<(), ContractError> {
    let key_manager = state
        .key_manager
        .as_ref()
        .ok_or(errors::ERR_KEY_MANAGER_IS_NOT_SET)?;
    if key_manager != predecessor_account_id {
        return Err(errors::ERR_NOT_ALLOWED.into());
    }
    Ok(())
}
