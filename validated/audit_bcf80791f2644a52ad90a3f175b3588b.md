### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped `setPoolBinAdditionalFees` - (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

### Summary

`setPoolBinAdditionalFees` forwards `addFeeBuyE6` / `addFeeSellE6` directly to the pool with no validation against `maxAdminSpreadFeeE6`, while the parallel `setPoolAdminFees` path enforces that cap. A pool admin can set per-bin additional fees up to `uint16` max (65 535 = 6.5535% in E6 units) regardless of what the factory owner has configured as the admin spread cap, breaking the admin-boundary invariant and causing traders to pay fees above the protocol-permitted ceiling.

### Finding Description

The factory enforces a configurable `maxAdminSpreadFeeE6` cap (set by the factory owner via `setFeeCaps`, bounded by `HARD_MAX_SPREAD_FEE_E6 = 200_000`). `setPoolAdminFees` correctly enforces this cap:

```solidity
// MetricOmmPoolFactory.sol line 414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

But `setPoolBinAdditionalFees` performs no such check — it passes the caller-supplied values straight through:

```solidity
// MetricOmmPoolFactory.sol line 450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool's `setBinAdditionalFees` also performs no cap check — it only validates the bin index: [3](#0-2) 

These per-bin fees are added directly on top of the base spread fee in every swap path:

```solidity
// MetricOmmPool.sol line 910
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)
``` [4](#0-3) [5](#0-4) [6](#0-5) 

`addFeeBuyE6` and `addFeeSellE6` are `uint16` fields, so the maximum settable value is 65 535, which equals 6.5535% in E6 units — well above any `maxAdminSpreadFeeE6` the factory owner may have configured (e.g., 0.1% = 1 000). [7](#0-6) 

### Impact Explanation

The factory owner's `setFeeCaps` is the mechanism by which the protocol enforces an upper bound on how much fee a pool admin can impose on traders. When `maxAdminSpreadFeeE6` is lowered below `uint16` max (e.g., to 1 000 = 0.1%), the pool admin can still call `setPoolBinAdditionalFees` with `addFeeBuyE6 = 65535` on every active bin, causing every swap through those bins to pay an effective spread fee of `baseFee + 6.5535%` — far above the protocol-permitted ceiling. Traders suffer direct, quantifiable overpayment on every swap; the excess fee accrues as LP surplus rather than being blocked. The factory owner's cap is rendered meaningless for the per-bin path. [8](#0-7) [9](#0-8) 

### Likelihood Explanation

Every pool has a pool admin who can call `setPoolBinAdditionalFees` at any time with no timelock. The call requires only `onlyPoolAdmin(pool)` — a semi-trusted role that is explicitly scoped to operate only within factory-configured caps. The bypass is a single transaction with no preconditions beyond holding the pool admin role. [10](#0-9) 

### Recommendation

Add the same cap check to `setPoolBinAdditionalFees` that `setPoolAdminFees` already applies. Because per-bin fees are additive on top of the global spread fee, the natural bound is that each bin's additional fee must not exceed `maxAdminSpreadFeeE6` independently:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
  external override nonReentrant onlyPoolAdmin(pool)
{
  if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
  IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce the cap inside `setBinAdditionalFees` on the pool itself by passing the cap as a parameter from the factory.

### Proof of Concept

1. Factory owner calls `setFeeCaps(maxProtocolSpreadFeeE6=5000, maxAdminSpreadFeeE6=1000, ...)` — capping admin spread at 0.1%.
2. Pool admin calls `setPoolAdminFees(pool, 1001, 0)` → reverts `AdminFeeTooHigh`. Cap is enforced.
3. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)` → **succeeds**. No cap check.
4. A trader swaps through bin 0. The effective buy fee is `baseFeeX64 + mulDiv(65535, ONE_X64, 1e6)` = base + 6.5535%, far above the 0.1% cap.
5. The trader pays 65× the permitted admin fee increment on every swap through that bin. [2](#0-1) [3](#0-2) [11](#0-10)
