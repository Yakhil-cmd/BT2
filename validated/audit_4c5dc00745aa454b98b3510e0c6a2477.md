### Title
Unrestricted `update_rewards` Allows Any Staker to Monopolize Per-Block Consensus Rewards - (File: src/staking/staking.cairo)

---

### Summary

The `update_rewards` function in the `Staking` contract is publicly callable with no access control. A single global `last_reward_block` variable enforces a one-call-per-block constraint. Any registered staker can call this function every block with their own address, receiving block rewards for every block while permanently blocking all other stakers from receiving any consensus rewards.

---

### Finding Description

`update_rewards` is exposed via the `IStakingRewardsManager` interface and is callable by any non-zero, non-paused address: [1](#0-0) 

The only gate is `general_prerequisites()`, which checks only that the contract is not paused and the caller is non-zero: [2](#0-1) 

The function enforces a single execution per block via a shared global variable: [3](#0-2) 

After passing the block check, it writes the current block number to `last_reward_block` and distributes block rewards to the caller-supplied `staker_address`: [4](#0-3) 

The `staker_address` parameter is entirely attacker-controlled — there is no check that the caller is the staker, the staker's reward address, or any privileged role. The interface confirms this is a fully open external entry point: [5](#0-4) 

Because `last_reward_block` is a single global slot, only **one** staker can receive consensus block rewards per block. Any staker who calls `update_rewards(their_own_address, false)` first in a block claims that block's rewards and sets `last_reward_block`, causing every other staker's call in the same block to revert with `REWARDS_ALREADY_UPDATED`. [6](#0-5) 

---

### Impact Explanation

An attacker who is a registered staker can call `update_rewards(attacker_staker, false)` in every block. Because `last_reward_block` is global and single-valued, this:

1. Grants the attacker's staker consensus block rewards for **every** block.
2. Permanently denies all other stakers their block rewards — their calls revert with `REWARDS_ALREADY_UPDATED` for every block the attacker occupies.

Missed block rewards are not retroactively recoverable; the per-block reward is distributed only to the staker named in the single successful call per block. This constitutes **theft of unclaimed yield** from all other stakers in the protocol.

**Impact: High** — Theft of unclaimed yield / permanent freezing of unclaimed yield for all non-attacking stakers.

---

### Likelihood Explanation

- The attacker only needs to be a registered staker (minimum stake, no special role).
- No leaked keys, bridge compromise, or privileged access is required.
- The attack is mechanically simple: submit `update_rewards(self, false)` every block.
- On Starknet, the sequencer orders transactions; a staker who submits this call at the start of every block will reliably win the race against other stakers who submit their own `update_rewards` calls.
- The attack is sustainable indefinitely at negligible cost (only transaction fees).

**Likelihood: High** — Any staker can execute this continuously with no barrier beyond minimum stake.

---

### Recommendation

Add an access-control check to `update_rewards` restricting callers to a trusted consensus role (e.g., `only_app_governor` or a dedicated consensus contract address), analogous to how `update_rewards_from_attestation_contract` is restricted to the attestation contract: [7](#0-6) 

Alternatively, redesign the per-block reward model so that each staker's rewards accumulate independently (removing the single-slot `last_reward_block` bottleneck), or require the caller to be the staker or their reward address.

---

### Proof of Concept

1. Attacker registers as a staker via `stake(reward_addr, op_addr, min_stake)`.
2. Every block, attacker submits `update_rewards(attacker_staker_address, false)`.
3. `last_reward_block` is set to the current block; attacker's staker receives that block's consensus rewards.
4. Honest staker Alice submits `update_rewards(alice_address, false)` in the same block — her call reverts with `REWARDS_ALREADY_UPDATED`.
5. Alice's block rewards for that block are permanently lost.
6. Repeated every block: attacker accumulates 100% of all consensus block rewards regardless of their proportional stake; all other stakers receive zero. [8](#0-7)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
