### Title
Unprivileged Caller Can Permanently Freeze Staker Consensus Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is publicly callable by any non-zero address and accepts an unvalidated `disable_rewards: bool` parameter. When set to `true`, the function advances the `last_reward_block` checkpoint but distributes zero rewards. Because the checkpoint is written before the guard check, any subsequent legitimate call for the same block reverts. An attacker can call this every block to permanently freeze a staker's consensus-era yield.

### Finding Description
`update_rewards` is exposed as a public ABI function under `StakingRewardsManagerImpl`. Its only gate is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero. [1](#0-0) 

Inside the function, `last_reward_block` is unconditionally written to the current block number **before** the `disable_rewards` branch is evaluated: [2](#0-1) 

Once `last_reward_block` equals the current block, the guard at line 1454–1458 causes every subsequent call in the same block to revert with `REWARDS_ALREADY_UPDATED`. The staker's operational address therefore cannot recover rewards for that block. [3](#0-2) 

There is no check that the caller is the staker, the staker's operational address, or any privileged role. The `disable_rewards` input is accepted from any caller without validation.

### Impact Explanation
Each attacker-controlled call with `disable_rewards: true` causes the targeted staker to permanently lose the block reward they would have earned for that block. Rewards in the consensus model are per-block; a missed block is never retroactively compensated. Repeated across every block, this constitutes permanent freezing (and effective theft) of unclaimed yield for the targeted staker. The impact maps to **High: Permanent freezing of unclaimed yield / Theft of unclaimed yield**.

### Likelihood Explanation
Medium. The attacker requires no special role, no capital, and no prior relationship with the staker. The only requirement is submitting a transaction before the staker's legitimate `update_rewards` call in each block. On Starknet, where the sequencer orders transactions, a motivated attacker monitoring the chain can reliably front-run this call. The gas cost per block is low, making sustained griefing economically feasible.

### Recommendation
Add caller validation to `update_rewards`. Only the staker's registered operational address (or the staker address itself) should be permitted to call this function. Alternatively, remove `disable_rewards` from the public interface entirely and handle the reward-skip logic through an internal path that is not externally triggerable.

// Example guard to add at the top of update_rewards:
let staker_info = self.internal_staker_info(:staker_address);

### Proof of Concept
1. Staker `S` is registered; their operational address is `O`.
2. Attacker `A` (any non-zero address) monitors the chain.
3. At each new Starknet block, `A` submits `update_rewards(S, disable_rewards: true)`.
4. The call passes `general_prerequisites()` (contract unpaused, caller non-zero).
5. `last_reward_block` is written to the current block number.
6. The `disable_rewards` branch fires; the function returns with zero rewards distributed.
7. When `O` subsequently calls `update_rewards(S, disable_rewards: false)`, it reverts with `REWARDS_ALREADY_UPDATED` because `current_block_number == last_reward_block`.
8. `S` earns zero consensus rewards for that block.
9. Repeated every block, `S` permanently accumulates no rewards despite being an active, fully-staked validator. [4](#0-3)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
