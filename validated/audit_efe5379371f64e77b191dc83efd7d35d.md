### Title
Unprivileged Caller Can Permanently Suppress Block Rewards via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract is publicly callable by any address and accepts a caller-controlled `disable_rewards: bool` parameter. A malicious actor can call it with `disable_rewards: true` in every block, consuming the global `last_reward_block` slot without distributing rewards, permanently denying all stakers their consensus-era block rewards.

### Finding Description
`IStakingRewardsManager::update_rewards` is exposed as a public entry point with no role-based access control. Its only gate is `general_prerequisites()`, which checks only that the contract is unpaused and the caller is non-zero. [1](#0-0) 

The function writes the current block number to the **single global** `last_reward_block` field unconditionally, before checking `disable_rewards`: [2](#0-1) 

`last_reward_block` is a single, non-per-staker storage slot: [3](#0-2) 

The guard that prevents double-processing compares against this global slot: [4](#0-3) 

Because the write to `last_reward_block` happens **before** the `disable_rewards` branch, any caller who front-runs with `disable_rewards: true` marks the block as "already processed" for every staker simultaneously, with zero rewards distributed.

### Impact Explanation
An attacker who calls `update_rewards(any_valid_staker, disable_rewards: true)` in every block:
- Marks each block as processed globally (`last_reward_block = current_block`)
- Causes the early return before any reward calculation or transfer
- Prevents every staker and every delegation pool from accruing consensus-era block rewards

This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators while the attack is sustained, matching the High-severity impact tier. The staker's `unclaimed_rewards_own` and pool `cumulative_rewards_trace` are never updated, so yield is silently lost rather than deferred. [5](#0-4) 

### Likelihood Explanation
- The function is fully public; no privileged key or role is required.
- The attacker only needs to know any valid, active staker address (all staker addresses are emitted as `NewStaker` events and stored in the public `stakers` vector).
- On Starknet, submitting one transaction per block is economically feasible; the attacker has no profit motive requirement — pure griefing suffices.
- The attack is front-running-based and requires sustained effort, but a single missed block by the attacker merely loses one block of rewards for the attacker's goal, not the attack itself.

### Recommendation
Restrict who may call `update_rewards` and what value they may supply for `disable_rewards`:

1. **Access control**: Require the caller to be either the staker's `operational_address` or a designated rewards-updater role. The `disable_rewards: true` path (used during migration/special cases) should be callable only by a privileged role (e.g., `app_governor` or `security_agent`).
2. **Separate the paths**: Split `update_rewards` into two functions — one for normal reward distribution (callable by the staker/operational address) and one for the disable path (callable only by governance).
3. **Per-staker tracking**: If the global `last_reward_block` must remain, gate the `disable_rewards: true` path behind `roles.only_app_governor()` or equivalent.

### Proof of Concept
1. Staker Alice stakes STRK and consensus rewards are active (`consensus_rewards_first_epoch` has passed).
2. In block `N`, attacker Bob calls `update_rewards(alice_address, disable_rewards: true)`.
   - `general_prerequisites()` passes (contract unpaused, Bob is non-zero).
   - `current_block_number (N) > last_reward_block` passes.
   - `last_reward_block` is written to `N`.
   - `disable_rewards == true` → early return; no rewards calculated or transferred.
3. Alice (or anyone) attempts `update_rewards(alice_address, disable_rewards: false)` in the same block `N`.
   - `current_block_number (N) > last_reward_block (N)` → **false** → reverts with `REWARDS_ALREADY_UPDATED`.
4. Alice loses all block rewards for block `N`. Bob repeats in block `N+1`, `N+2`, … permanently suppressing all staker rewards at the cost of one cheap transaction per block. [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
