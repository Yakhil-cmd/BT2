### Title
Pause State Not Enforced in `staker_migration` — Authorization Bypass Allows Pool Upgrades While Contract Is Paused - (File: src/staking/staking.cairo)

### Summary
The `staker_migration` function in the Staking contract does not call `general_prerequisites()`, which is the function that enforces the pause check. As a result, any caller can trigger pool contract upgrades even when the protocol is in an emergency paused state, bypassing the intended freeze of all state-changing operations.

### Finding Description
Every state-mutating function in the Staking contract calls `self.general_prerequisites()` at its entry point, which enforces the `is_paused` flag. For example, `stake`, `unstake_intent`, `add_stake_from_pool`, `remove_from_delegation_pool_intent`, `set_commission`, `set_public_key`, and all other user-facing functions follow this pattern.

However, `staker_migration` (lines 956–998 of `src/staking/staking.cairo`) does not call `general_prerequisites()`:

```cairo
fn staker_migration(ref self: ContractState, staker_address: ContractAddress) {
    // Assert the staker exists.
    self._internal_staker_info(:staker_address);
    // Assert staker already migrated to V2.
    assert!(...);
    // Assert the staker is not migrated to V3 yet.
    assert!(...);
    // ... upgrades pool contracts via add_new_implementation + replace_to
    self.staker_version.write(staker_address, LATEST_STAKER_VERSION);
}
``` [1](#0-0) 

The function is permissionless — any caller can supply any `staker_address` — and it performs two irreversible state changes:
1. Calls `add_new_implementation` and `replace_to` on each of the staker's pool contracts, upgrading them to the V3 class hash stored in `pool_contract_class_hash`.
2. Writes `LATEST_STAKER_VERSION` to `staker_version` for the staker, permanently marking them as migrated. [2](#0-1) 

By contrast, every other state-mutating function enforces the pause: [3](#0-2) [4](#0-3) 

### Impact Explanation
The pause mechanism is the protocol's emergency brake. If a critical vulnerability is discovered in the V3 pool contract class hash that is already loaded into `pool_contract_class_hash`, operators would pause the contract to halt all state changes while a fix is prepared. During this window, an attacker (or any caller) can still invoke `staker_migration` for every staker, forcibly upgrading all pool contracts to the vulnerable V3 implementation. This permanently marks those stakers as migrated (`staker_version = V3`) and replaces their pool contract logic, potentially exposing all delegated funds held in those pools to the vulnerability the pause was meant to contain.

This maps to **Temporary (or Permanent) Freezing of Funds** — delegated funds in upgraded pool contracts could become inaccessible or exploitable if the upgrade was the attack vector the pause was guarding against.

### Likelihood Explanation
- The function is permissionless; no special role is required.
- The attacker only needs to know staker addresses (publicly readable from the `stakers` Vec in storage).
- The window of opportunity is exactly the pause period — the time between an operator detecting a critical bug and deploying a fix.
- The precondition (staker migrated to V2 but not yet V3) is the normal state for all stakers during the V2→V3 upgrade window, which is the exact scenario where this bypass is most dangerous.

### Recommendation
Add `self.general_prerequisites()` as the first statement in `staker_migration`, consistent with every other state-mutating function in the contract:

```cairo
fn staker_migration(ref self: ContractState, staker_address: ContractAddress) {
    self.general_prerequisites(); // ADD THIS
    self._internal_staker_info(:staker_address);
    // ...
}
``` [5](#0-4) 

### Proof of Concept
1. Operators discover a critical bug in the V3 pool contract and call `pause()` on the Staking contract. `is_paused` is set to `true`.
2. Attacker reads the `stakers` Vec from storage to enumerate all staker addresses.
3. For each staker address `s` that is in V2 state (migrated to V2 but not V3), attacker calls `staker_migration(s)`.
4. `general_prerequisites()` is never called; the pause flag is never checked.
5. Each staker's pool contracts are upgraded to the vulnerable V3 class hash via `replace_to`, and `staker_version[s]` is set to `V3`.
6. The pause no longer protects delegated funds in those pools — the vulnerable code is now live. [6](#0-5)

        fn staker_migration(ref self: ContractState, staker_address: ContractAddress) {
            // Assert the staker exists.
            self._internal_staker_info(:staker_address);

            // Assert staker already migrated to V2.
            assert!(
                !self.staker_own_balance_trace.entry(staker_address).is_empty(),
                "{}",
                Error::STAKER_NOT_MIGRATED,
            );

            // Assert the staker is not migrated to V3 yet.
            assert!(
                !self.staker_version.read(staker_address).is_latest(),
                "{}",
                Error::STAKER_ALREADY_MIGRATED,
            );

            // Prepare the implementation data.
            let pool_class_hash = self.pool_contract_class_hash.read();
            let pool_eic_class_hash = self.pool_eic_class_hash.read();
            // Sanity checks.
            assert!(pool_class_hash.is_non_zero(), "{}", InternalError::MISSING_CLASS_HASH);
            assert!(pool_eic_class_hash.is_non_zero(), "{}", InternalError::MISSING_CLASS_HASH);
            let eic_data = EICData { eic_hash: pool_eic_class_hash, eic_init_data: [].span() };
            let implementation_data = ImplementationData {
                impl_hash: pool_class_hash, eic_data: Option::Some(eic_data), final: false,
            };

            // Upgrade pools.
            let pools = self.staker_pool_info.entry(staker_address).pools;
            for pool_contract_ptr in pools.keys_iter() {
                let pool_contract = pool_contract_ptr.read();
                let pool_replaceable_dispatcher = IReplaceableDispatcher {
                    contract_address: pool_contract,
                };
                pool_replaceable_dispatcher.add_new_implementation(:implementation_data);
                pool_replaceable_dispatcher.replace_to(:implementation_data);
            }

            // Mark the staker's version.
            self.staker_version.write(staker_address, LATEST_STAKER_VERSION);
        }
