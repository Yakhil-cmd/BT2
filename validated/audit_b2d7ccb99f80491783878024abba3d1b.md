### Title
Unprivileged Caller Can Permanently Freeze Consensus Rewards via `disable_rewards` Flag in `update_rewards` ‚Äî (File: `src/staking/staking.cairo`)

---

### Summary

The `update_rewards` function in the `Staking` contract uses a **global** `last_reward_block` to enforce a single reward distribution per block. The function accepts a caller-controlled `disable_rewards: bool` parameter with no access control. Critically, `last_reward_block` is written to storage **before** the `disable_rewards` check, meaning any unprivileged caller can consume the per-block reward slot without distributing any rewards. By calling `update_rewards(any_valid_staker, disable_rewards: true)` every block, an attacker permanently freezes all stakers' consensus (V3) rewards at zero cost beyond gas.

---

### Finding Description

`update_rewards` is a public external function in `StakingRewardsManagerImpl`:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks: not paused, caller != 0
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);   // ‚Üê slot consumed HERE

    if disable_rewards || self.is_pre_consensus() {
        return;                                            // ‚Üê exits WITHOUT distributing rewards
    }
    ...
``` [1](#0-0) 

The `general_prerequisites()` check only asserts the contract is unpaused and the caller is non-zero ‚Äî no role or identity check is performed. [2](#0-1) 

The global `last_reward_block` is written at line 1485 **before** the `disable_rewards` branch at line 1487. Once written, no other call to `update_rewards` can succeed in the same block (the `REWARDS_ALREADY_UPDATED` assertion at line 1455 will revert). [3](#0-2) 

**Attack path:**

1. Attacker (any non-zero address, no staker role required) calls `update_rewards(any_valid_active_staker, disable_rewards: true)` on the first transaction of every block.
2. The function passes all checks (staker exists, has non-zero balance, block is new).
3. `last_reward_block` is set to the current block number.
4. The function returns early ‚Äî zero rewards are distributed.
5. Every subsequent legitimate call to `update_rewards` in that block reverts with `REWARDS_ALREADY_UPDATED`.
6. Repeated every block, all consensus rewards are permanently frozen.

The attacker only needs to supply a valid `staker_address` (any active staker from the public `stakers` vector). The attacker does not need to be a staker themselves.

---

### Impact Explanation

All stakers operating under the V3 consensus reward regime (`is_pre_consensus() == false`) are permanently denied their block rewards. The `unclaimed_rewards_own` field in each staker's `InternalStakerInfoLatest` never increases, and pool rewards are never forwarded to delegation pools. This constitutes **permanent freezing of unclaimed yield** for all stakers and delegators in the protocol. [4](#0-3) 

---

### Likelihood Explanation

- The function is fully public with no role guard.
- The attacker needs only a non-zero address and enough gas to call once per block.
- Gas costs on Starknet L2 are low, making sustained griefing economically viable.
- A competing staker has a direct financial incentive: by blocking all other stakers' rewards while claiming their own on alternating blocks, they increase their relative share of total rewards.
- The `stakers` vector is public, so any valid `staker_address` argument is trivially discoverable on-chain. [5](#0-4) 

---

### Recommendation

Move `last_reward_block.write(current_block_number)` to **after** the `disable_rewards` guard, so that consuming the per-block slot only occurs when rewards are actually being distributed:

```cairo
// Update last block rewards ONLY when rewards will be distributed.
if disable_rewards || self.is_pre_consensus() {
    return;
}
self.last_reward_block.write(current_block_number);
// ... proceed with reward calculation
```

Alternatively, restrict the `disable_rewards: true` path to a privileged role (e.g., `only_security_agent`), or remove the parameter entirely if it serves no legitimate unprivileged use case.

---

### Proof of Concept

```
Block N:
  Attacker calls: staking.update_rewards(staker=<any_valid_staker>, disable_rewards=true)
  ‚Üí general_prerequisites() passes (attacker is non-zero, contract unpaused)
  ‚Üí assert!(block_N > last_reward_block) passes (last_reward_block < N)
  ‚Üí staker checks pass (valid active staker with non-zero balance)
  ‚Üí last_reward_block.write(block_N)   ‚Üê slot consumed
  ‚Üí disable_rewards == true ‚Üí return   ‚Üê no rewards distributed

  Legitimate staker calls: staking.update_rewards(staker=<self>, disable_rewards=false)
  ‚Üí assert!(block_N > last_reward_block) REVERTS (last_reward_block == block_N)

Block N+1: attacker repeats ‚Üí all stakers receive 0 consensus rewards indefinitely.
``` [6](#0-5)

    impl StakingRewardsManagerImpl of IStakingRewardsManager<ContractState> {
        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
