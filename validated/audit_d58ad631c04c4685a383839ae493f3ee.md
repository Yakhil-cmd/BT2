### Title
Missing Caller Restriction on `update_rewards` Allows Anyone to Permanently Freeze Staker Yield - (File: `src/staking/staking.cairo`)

### Summary

`IStakingRewardsManager::update_rewards` is specified to be callable only by the Starkware sequencer, but the implementation contains no caller check. Any unprivileged address can call it with `disable_rewards: true` to advance the global `last_reward_block` sentinel without distributing any rewards. Because the function enforces a per-block uniqueness guard (`REWARDS_ALREADY_UPDATED`), the legitimate sequencer is then blocked from distributing rewards for that block. Repeated calls every block permanently freeze all staker and delegator yield.

### Finding Description

The spec at `docs/spec.md:1644-1645` states:

> **access control**: Only starkware sequencer.

The implementation at `src/staking/staking.cairo:1449-1507` enforces no such restriction:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();                          // only checks pause
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    // ... NO caller check ...
    self.last_reward_block.write(current_block_number);   // global sentinel advanced
    if disable_rewards || self.is_pre_consensus() {
        return;                                           // exits without distributing
    }
    // reward distribution only reached if caller cooperates
}
```

`last_reward_block` is a **global** (not per-staker) storage variable. Once it is written to `current_block_number`, any subsequent call in the same block — including the legitimate sequencer call — reverts with `REWARDS_ALREADY_UPDATED`.

A grep for `only_sequencer` / `assert_caller_is_sequencer` across the entire repository returns zero matches in production code; the restriction exists only in documentation.

**Attack path:**

1. Attacker monitors the mempool / block production.
2. At each new block, attacker calls `update_rewards(any_valid_active_staker, disable_rewards: true)`.
3. `last_reward_block` is set to the current block number; no rewards are distributed.
4. The legitimate sequencer's `update_rewards` call for the same block reverts with `REWARDS_ALREADY_UPDATED`.
5. Rewards for that block are permanently lost — there is no catch-up mechanism.
6. Repeating this every block permanently freezes all consensus-era yield for every staker and delegator.

The attacker needs only a valid `staker_address` (any active staker, publicly readable from events) and the ability to submit transactions — no privileged access required.

### Impact Explanation

**Permanent freezing of unclaimed yield** for all stakers and all delegation pool members. Once a block's reward slot is consumed by the attacker's no-op call, the rewards for that block are irrecoverably lost. There is no retroactive recalculation. This matches the allowed High impact: *Permanent freezing of unclaimed yield or unclaimed royalties*.

### Likelihood Explanation

**High.** The entry point is a public, permissionless function. Starknet transaction fees are low. The attacker needs no stake, no special role, and no prior setup — only a valid staker address (trivially obtained from `NewStaker` events). The attack is sustainable indefinitely.

### Recommendation

Add an explicit caller check inside `update_rewards` that restricts execution to the authorized sequencer address, consistent with the specification. For example:

```cairo
fn update_rewards(...) {
    self.general_prerequisites();
    self.assert_caller_is_sequencer(); // enforce spec requirement
    ...
}
```

Alternatively, store the sequencer address in contract storage during initialization and assert `get_caller_address() == self.sequencer_address.read()` at the top of the function.

### Proof of Concept

```
Block N arrives.

Attacker tx (submitted first):
  staking.update_rewards(staker_address=<any_valid_staker>, disable_rewards=true)
  → last_reward_block := N
  → returns without distributing rewards

Sequencer tx (submitted same block):
  staking.update_rewards(staker_address=<staker>, disable_rewards=false)
  → assert!(N > last_reward_block.read())  // N > N is false
  → PANIC: REWARDS_ALREADY_UPDATED

Rewards for block N: permanently lost.
Repeat every block → all consensus-era yield frozen indefinitely.
```
