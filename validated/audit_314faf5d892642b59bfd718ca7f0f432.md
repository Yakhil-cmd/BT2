### Title
Missing Caller Restriction on `update_rewards` Allows Any Address to Permanently Block Reward Distribution - (File: src/staking/staking.cairo)

### Summary

`update_rewards` in the Starknet Staking contract enforces no caller access control. Any non-zero address can call it with `disable_rewards: true` to consume the single per-block reward slot (tracked by the global `last_reward_block`) without distributing any rewards. Because the slot is consumed, the legitimate sequencer's subsequent call for the same block reverts with `REWARDS_ALREADY_UPDATED`. Repeating this every block permanently freezes all staker and delegator unclaimed yield.

### Finding Description

`update_rewards` is the sole entry point for distributing consensus-era block rewards to stakers and their pools. The spec explicitly states its access control is "Only starkware sequencer," but the implementation enforces no such restriction.

`general_prerequisites()` only checks that the contract is not paused and the caller is not the zero address:

```cairo
fn general_prerequisites(ref self: ContractState) {
    self.assert_is_unpaused();
    assert_caller_is_not_zero();
}
``` [1](#0-0) 

`update_rewards` itself adds no `get_caller_address()` check:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
    assert!(
        current_block_number > self.last_reward_block.read(),
        "{}",
        Error::REWARDS_ALREADY_UPDATED,
    );
    ...
    // Update last block rewards.
    self.last_reward_block.write(current_block_number);

    if disable_rewards || self.is_pre_consensus() {
        return;
    }
    ...
``` [2](#0-1) 

`last_reward_block` is a **global** (not per-staker) storage variable: [3](#0-2) 

The spec confirms the intended restriction: [4](#0-3) 

### Impact Explanation

An attacker calls `update_rewards(staker_address: <any_active_staker>, disable_rewards: true)` at the start of every block. This:

1. Passes all checks (non-zero caller, block is new, staker is active).
2. Writes `current_block_number` into `last_reward_block`.
3. Returns immediately without distributing any rewards (because `disable_rewards == true`).

When the legitimate sequencer then calls `update_rewards` for the same block, it hits:

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
``` [5](#0-4) 

and reverts. No staker or delegator receives rewards for that block. Repeated every block, this **permanently freezes all unclaimed yield** for every staker and every delegation pool in the protocol.

This maps to the allowed impact: **High — Permanent freezing of unclaimed yield**.

### Likelihood Explanation

- The entry point is fully public (any non-zero address).
- The attacker needs only a valid active staker address (publicly readable via `get_stakers`).
- No capital, no privileged role, and no special setup is required.
- The cost is one Starknet transaction per block, which is negligible.
- The attack is sustainable indefinitely.

### Recommendation

Add a caller check inside `update_rewards` to restrict it to the authorized sequencer address (or the attestation contract, consistent with the V2 path). For example:

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    assert!(
        get_caller_address() == self.sequencer_address.read(),
        "{}",
        Error::CALLER_IS_NOT_SEQUENCER,
    );
    ...
```

Alternatively, store the authorized caller address during construction and enforce it here, mirroring the pattern used in `update_rewards_from_attestation_contract` which correctly restricts its caller to the attestation contract. [6](#0-5) 

### Proof of Concept

```
1. Deploy the system and start consensus rewards.
2. Advance K epochs so a staker has effective balance.
3. From any arbitrary address (attacker):
     staking.update_rewards(staker_address: <any_active_staker>, disable_rewards: true)
   → succeeds; last_reward_block = current_block.
4. From the legitimate sequencer, same block:
     staking.update_rewards(staker_address: <staker>, disable_rewards: false)
   → panics with REWARDS_ALREADY_UPDATED.
5. Advance one block; repeat step 3 from the attacker.
6. After N blocks: staker.unclaimed_rewards_own == 0 despite N blocks having elapsed.
```

The existing test suite already demonstrates the `REWARDS_ALREADY_UPDATED` revert behavior when called twice in the same block: [7](#0-6)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }
