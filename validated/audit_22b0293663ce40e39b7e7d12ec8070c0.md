### Title
Gateway Stateful Validator Skips L1/L1-Data Gas Price Threshold Check, Admitting Transactions That Will Fail Blockifier Pre-Validation - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

`StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold` enforces a minimum gas-price threshold only for the **L2 gas** component of `AllResources` transactions and explicitly skips all validation for `L1Gas` (legacy) transactions. Neither the L1 gas price nor the L1 data gas price is compared against the previous block's price at gateway admission time. As a result, any `AllResources` transaction whose `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit` is below the current block's L1/L1-data gas price passes both the stateless and stateful gateway validators, is admitted to the mempool, and is only rejected later—inside `AccountTransaction::check_fee_bounds`—during blockifier pre-validation.

### Finding Description

**Stateless validator** (`crates/apollo_gateway/src/stateless_transaction_validator.rs`, lines 56–88) checks only `resource_bounds.l2_gas.max_price_per_unit` against the static `min_gas_price` config. No check is performed on `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. [1](#0-0) 

**Stateful validator** (`crates/apollo_gateway/src/stateful_transaction_validator.rs`, lines 358–390) fetches the previous block's L2 gas price and calls `validate_tx_l2_gas_price_within_threshold`. Inside that function the match arm for `ValidResourceBounds::AllResources` checks only `l2_gas.max_price_per_unit`; the `ValidResourceBounds::L1Gas` arm is a no-op with the comment "No validation required for legacy transactions." The developer TODO at line 358 explicitly acknowledges the gap: *"Consider running this validation for all gas prices."* [2](#0-1) 

**Blockifier pre-validation** (`crates/blockifier/src/transaction/account_transaction.rs`, lines 374–475) does check all three gas prices against the current block's prices inside `check_fee_bounds`. For `AllResources` transactions it iterates over L1 gas, L1 data gas, and L2 gas and returns `ResourceBoundsError::MaxGasPriceTooLow` if any `max_price_per_unit` is below the actual block price. [3](#0-2) 

The gap is therefore: the gateway's stateful validator only enforces the L2 gas price floor; L1 and L1-data gas prices are unchecked at admission. A transaction that passes gateway admission can still be rejected by the blockifier, meaning the gateway's admission decision is wrong.

### Impact Explanation

This matches the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

A transaction with `l1_gas.max_price_per_unit = 0` (or any value below the current L1 gas price) and a valid `l2_gas.max_price_per_unit` passes both the stateless and stateful validators and is inserted into the mempool. When the batcher dequeues it and calls `AccountTransaction::perform_pre_validation_stage`, `check_fee_bounds` raises `InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } }` and the transaction is discarded. The gateway's admission decision is therefore incorrect: it accepted a transaction that the blockifier will unconditionally reject.

At scale an adversary can flood the mempool with such transactions at zero cost (no fee is charged for pre-validation failures), consuming mempool capacity and batcher CPU while producing no useful blocks.

### Likelihood Explanation

L1 gas prices fluctuate continuously. Any user who submits an `AllResources` transaction with `l1_gas.max_price_per_unit` set to zero (or to a stale low value) will have their transaction admitted by the gateway and silently dropped by the batcher. This is reachable by any unprivileged caller via the normal `add_tx` RPC path.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or add a parallel helper) to also compare `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the corresponding prices from the previous block's `BlockInfo`. The previous block already exposes `gas_prices.strk_gas_prices.l1_gas_price` and `gas_prices.strk_gas_prices.l1_data_gas_price` through the same `get_block_info()` call that is already made for the L2 price check. Applying the same `min_gas_price_percentage` threshold to all three resources would make the gateway's admission decision consistent with the blockifier's pre-validation check.

### Proof of Concept

1. Construct an `AllResources` invoke transaction with:
   - `l2_gas.max_price_per_unit` ≥ `min_gas_price` (passes stateless check)
   - `l1_gas.max_price_per_unit = 0` (below any real L1 gas price)
   - `l1_data_gas.max_price_per_unit = 0`

2. Submit via `add_tx` RPC.

3. **Stateless validator** (`validate_resource_bounds`): passes — only `l2_gas.max_price_per_unit` is compared to `min_gas_price`. [1](#0-0) 

4. **Stateful validator** (`validate_tx_l2_gas_price_within_threshold`): passes — only `l2_gas.max_price_per_unit` is compared to the previous block's L2 price; L1 and L1-data prices are not checked. [4](#0-3) 

5. Transaction is inserted into the mempool.

6. Batcher dequeues the transaction and calls `AccountTransaction::perform_pre_validation_stage` → `check_fee_bounds`. The L1 gas check fires:
   ```
   resource_bounds.max_price_per_unit (0) < actual_gas_price.get() (e.g. 8_000_000_000)
   → ResourceBoundsError::MaxGasPriceTooLow { resource: L1Gas, … }
   → TransactionFeeError::InsufficientResourceBounds
   → TransactionPreValidationError
   ``` [5](#0-4) 

7. Transaction is discarded by the batcher. The gateway's admission decision was wrong: it accepted a transaction the blockifier unconditionally rejects.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
