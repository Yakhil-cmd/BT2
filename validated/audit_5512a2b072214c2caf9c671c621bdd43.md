### Title
Malicious Participant Can Corrupt CKD Output via Unverified `app_pk` Input — (File: `src/confidential_key_derivation/protocol.rs`)

---

### Summary

The `ckd()` function accepts `app_pk` as an unchecked, caller-supplied input with no cross-participant consistency verification. A malicious participant can substitute an arbitrary `app_pk'` in their local `compute_signature_share` call, causing the aggregated `CKDOutput` to be permanently corrupted and unrecoverable by the legitimate application.

---

### Finding Description

**Vulnerability class**: Arbitrary input accepted without identity/ownership verification (direct analog of the DaosLocker `collect(address dao)` pattern).

In `compute_signature_share` (`src/confidential_key_derivation/protocol.rs`, lines 148–182), each participant computes their ElGamal-encrypted share contribution as:

```rust
let big_c = big_s + app_pk * y.0;   // line 174
```

where `big_s = x_i * H(pk || app_id)` is the participant's secret share contribution and `y` is a fresh random scalar. [1](#0-0) 

The `app_pk` parameter flows in from the public `ckd()` entry point with no validation beyond what the caller provides:

```rust
pub fn ckd(
    participants: &[Participant],
    coordinator: Participant,
    me: Participant,
    key_pair: KeygenOutput,
    app_id: impl Into<AppId>,
    app_pk: PublicKey,          // ← unchecked caller input
    rng: impl CryptoRngCore + Send + 'static,
) -> Result<impl Protocol<Output = CKDOutputOption>, InitializationError>
``` [2](#0-1) 

The coordinator aggregates all participants' `(norm_big_y, norm_big_c)` values without any verification of which `app_pk` each participant used:

```rust
for (_, participant_output) in
    recv_from_others::<CKDOutput>(&chan, waitpoint, &participants, me).await?
{
    norm_big_y += participant_output.big_y();
    norm_big_c += participant_output.big_c();
}
``` [3](#0-2) 

There is no broadcast or echo-broadcast of `app_pk` among participants to establish consistency — unlike `session_id` in `do_keyshare`, which is explicitly broadcast so all parties agree on it. [4](#0-3) 

**Mathematical consequence**: If malicious participant `m` uses `app_pk' ≠ app_pk`, the aggregated output becomes:

```
C = msk·H(pk ∥ app_id) + Σ_{i≠m}(λ_i·y_i·app_pk) + λ_m·y_m·app_pk'
```

When the legitimate application calls `unmask(a)`:

```rust
pub fn unmask(&self, secret_scalar: Scalar) -> Signature {
    self.big_c - self.big_y * secret_scalar   // C − a·Y
}
``` [5](#0-4) 

The result is:

```
C − a·Y = msk·H(pk ∥ app_id) + λ_m·y_m·(app_pk' − app_pk)
```

The residual term `λ_m·y_m·(app_pk' − app_pk)` is non-zero and unknown to the application, making the derived key permanently unrecoverable for this session.

---

### Impact Explanation

This matches **High: Corruption of CKD outputs so honest parties accept unusable cryptographic outputs**. The legitimate application receives a `CKDOutput` that cannot be correctly unmasked via `CKDOutput::unmask`. Since the malicious participant can repeat this attack on every CKD invocation, it constitutes a **permanent denial of CKD** for honest parties under valid protocol inputs. The attack is silent — the coordinator and honest participants have no mechanism to detect the substitution because `app_pk` is never broadcast or verified across participants.

---

### Likelihood Explanation

Any single malicious participant in the CKD participant set can execute this attack unilaterally. No coordination with other participants is required. The attacker simply passes a different `app_pk` to their local `ckd()` call. The attack requires no cryptographic capability beyond choosing an arbitrary group element. The library provides no detection or attribution mechanism for this deviation.

---

### Recommendation

Add a broadcast/echo-broadcast of `app_pk` among all participants at the start of the CKD protocol, analogous to how `session_id` is broadcast in `do_keyshare`:

```rust
// In run_ckd_protocol, before compute_signature_share:
let agreed_app_pk = do_broadcast(&mut chan, &participants, me, app_pk).await?;
// Verify all participants broadcast the same app_pk
for (p, their_pk) in agreed_app_pk.iter() {
    if their_pk != &app_pk {
        return Err(ProtocolError::MaliciousParticipant(p));
    }
}
```

Alternatively, include `app_pk` in the session transcript hash so that any deviation is cryptographically detectable and attributable.

---

### Proof of Concept

1. Honest participants `{1, 2, 3}` run `ckd()` with legitimate `app_pk = a·G`.
2. Malicious participant `2` calls `ckd()` with `app_pk' = a'·G` where `a' ≠ a`.
3. Participant `2`'s contribution: `C_2 = λ_2·(x_2·H(pk ∥ app_id) + y_2·app_pk')`.
4. Coordinator aggregates (lines 50–56): `C = msk·H(pk ∥ app_id) + λ_1·y_1·app_pk + λ_2·y_2·app_pk' + λ_3·y_3·app_pk`.
5. Application calls `unmask(a)` (line 55): result = `msk·H(pk ∥ app_id) + λ_2·y_2·(app_pk' − app_pk)` ≠ `msk·H(pk ∥ app_id)`.
6. The derived key is permanently corrupted for this and every subsequent CKD session involving participant `2`. [6](#0-5)

fn compute_signature_share(
    participants: &ParticipantList,
    me: Participant,
    key_pair: &KeygenOutput,
    app_id: &AppId,
    app_pk: PublicKey,
    rng: &mut impl CryptoRngCore,
) -> Result<(ElementG1, ElementG1), ProtocolError> {
    // Ensures the value is zeroized on drop
    let private_share = Zeroizing::new(key_pair.private_share);

    // y <- ZZq* , Y <- y * G
    let y = Scalar::random(rng);

    // Ensures the value is zeroized on drop
    let y = Zeroizing::new(super::scalar_wrapper::ScalarWrapper(y));

    let big_y = ElementG1::generator() * y.0;

    // H(pk || app_id) when H is a random oracle
    let hash_point = hash_app_id_with_pk(&key_pair.public_key, app_id);

    // S <- x . H(app_id)
    let big_s = hash_point * private_share.to_scalar();

    // C <- S + y . A
    let big_c = big_s + app_pk * y.0;

    // Compute  λi := λi(0)
    let lambda_i = participants.lagrange::<BLS12381SHA256>(me)?;
    // Normalize Y and C into  (λi . Y , λi . C)
    let norm_big_y = big_y * lambda_i;
    let norm_big_c = big_c * lambda_i;
    Ok((norm_big_y, norm_big_c))
}
