### Title
Increasing `attestation_window` Mid-Epoch Allows Stakers Who Missed Their Window to Backrun and Claim Unearned Rewards - (File: src/attestation/attestation.cairo)

---

### Summary

`set_attestation_window` takes effect immediately in the current epoch and recalculates `target_attestation_block` for every staker. A staker who has already passed their old attestation window end can backrun the admin's `set_attestation_window` call to find themselves inside the newly computed window and claim epoch rewards they should not have earned.

---

### Finding Description

The `_calculate_target_attestation_block` function derives each staker's target block from the live `attestation_window` storage value:

```
block_offset = hash % (epoch_len - attestation_window)
target       = epoch_start + block_offset
```

The valid attestation range is then `[target + MIN_ATTESTATION_WINDOW, target + attestation_window]`. [1](#0-0) 

Both the target block and the valid range are recomputed on every call to `attest()` using whatever `attestation_window` is stored at that moment. [2](#0-1) 

`set_attestation_window` writes the new value immediately with no epoch-boundary guard: [3](#0-2) 

The interface explicitly acknowledges the immediate effect but only warns about validators *missing* rewards, not about validators *gaining* rewards they should not have: [4](#0-3) 

**Concrete scenario**

| Variable | Value |
|---|---|
| `epoch_len` | 100 blocks |
| `epoch_start` | block 0 |
| `old_window` | 20 |
| Staker Poseidon hash | 50 |
| Old target | `0 + (50 % 80)` = block 50 |
| Old valid range | [61, 70] |
| Current block | 75 — staker has **missed** their window |

Admin calls `set_attestation_window(80)`:

| Variable | Value |
|---|---|
| New target | `0 + (50 % 20)` = block 10 |
| New valid range | [21, 90] |
| Current block 75 | **inside** [21, 90] ✓ |

The staker can now call `attest(block_hash_of_block_10)` — block 10 is in the past, so its hash is available via `get_block_hash_syscall` — and successfully claim epoch rewards. [5](#0-4) 

---

### Impact Explanation

A staker who legitimately missed their attestation window can backrun a governance `set_attestation_window` increase to attest and receive epoch rewards they should not have earned. Epoch rewards are distributed proportionally to staking power among all attesters; an illegitimate extra attester dilutes every other staker's share of that epoch's minted rewards. This constitutes **theft of unclaimed yield** (High impact).

---

### Likelihood Explanation

- `set_attestation_window` is a governance action that will be called periodically (e.g., to widen the window for operational reasons).
- Whether the new window puts a specific staker back in range is deterministic and publicly computable from on-chain data (`hash`, `epoch_len`, new window size).
- A staker who missed their window can monitor for `set_attestation_window` transactions, pre-compute the new target block, and submit `attest()` in the immediately following block.
- No special privileges are required beyond controlling one's own operational address.

Likelihood: **Medium** (requires a governance window-increase that happens to shift a staker's target into the current block range, but this is predictable and exploitable by any staker who missed their window).

---

### Recommendation

Make `set_attestation_window` take effect only at the start of the **next** epoch. Store a pending value alongside the current value and apply it when the epoch advances. This mirrors the fix in the referenced Oracle report: state that was accumulated under the old parameter must not be retroactively reinterpreted under the new one.

Alternatively, record the `attestation_window` value that was in effect at the start of each epoch and use that frozen value for all attestation validation within that epoch, regardless of mid-epoch governance changes.

---

### Proof of Concept

```
Epoch: length = 100 blocks, starts at block 0
Staker: Poseidon(stake, epoch_id, staker_address) = 50
attestation_window (initial) = 20

Step 1 — Staker misses their window:
  target_old = 0 + (50 % (100 - 20)) = 0 + (50 % 80) = 50
  valid range = [50+11, 50+20] = [61, 70]
  current_block = 75  →  attest() reverts with ATTEST_OUT_OF_WINDOW

Step 2 — Admin calls set_attestation_window(80):
  attestation_window is now 80

Step 3 — Staker backruns in the next block (still block 75):
  target_new = 0 + (50 % (100 - 80)) = 0 + (50 % 20) = 10
  valid range = [10+11, 10+80] = [21, 90]
  current_block = 75  →  75 ∈ [21, 90]  ✓

Step 4 — Staker calls attest(get_block_hash(block 10)):
  Block 10 is in the past; its hash is available on-chain.
  _validate_attestation passes all checks.
  update_rewards_from_attestation_contract is called.
  Staker receives epoch rewards they should not have earned.
``` [6](#0-5) [7](#0-6)

        fn set_attestation_window(ref self: ContractState, attestation_window: u16) {
            self.roles.only_app_governor();
            assert!(
                attestation_window >= MIN_ATTESTATION_WINDOW, "{}", Error::ATTEST_WINDOW_TOO_SMALL,
            );
            let old_attestation_window = self.attestation_window.read();
            self.attestation_window.write(attestation_window);
            self
                .emit(
                    Events::AttestationWindowChanged {
                        old_attestation_window, new_attestation_window: attestation_window,
                    },
                );
        }

        fn _calculate_target_attestation_block(
            self: @ContractState, staking_attestation_info: StakingAttestationInfo,
        ) -> BlockNumber {
            // Compute staker hash for the attestation.
            let hash = PoseidonTrait::new()
                .update(staking_attestation_info.stake().into())
                .update(staking_attestation_info.epoch_id().into())
                .update(staking_attestation_info.staker_address().into())
                .finalize();
            // Calculate staker's block number in this epoch.
            let attestation_window = self.attestation_window.read();
            let block_offset: u256 = hash
                .into() % (staking_attestation_info.epoch_len() - attestation_window.into())
                .into();
            // Calculate actual block number for attestation.
            let target_attestation_block = staking_attestation_info.current_epoch_starting_block()
                + block_offset.try_into().unwrap();
            target_attestation_block
        }

        fn _assert_attest_in_window(self: @ContractState, target_attestation_block: BlockNumber) {
            let attestation_window = self.attestation_window.read();
            let current_block_number = get_block_number();
            let min_block = target_attestation_block + MIN_ATTESTATION_WINDOW.into();
            let max_block = target_attestation_block + attestation_window.into();
            assert!(
                min_block <= current_block_number && current_block_number <= max_block,
                "{}",
                Error::ATTEST_OUT_OF_WINDOW,
            );
        }
