### Title
Gateway Stateful Validator Uses Stale `l2_gas_price` Instead of `next_l2_gas_price` for Resource-Bounds Admission — (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

`StatefulTransactionValidator::validate_resource_bounds` computes its admission threshold from `previous_block.l2_gas_price` — the price that was used for transactions **already in the last finalized block** — rather than `previous_block.next_l2_gas_price`, which is the EIP-1559-derived price that will actually govern the **next** block. The mismatch causes the gateway to reject valid transactions (falling-price market) or admit transactions that will fail at blockifier execution (rising-price market). A developer-authored TODO in the same line of code explicitly acknowledges the wrong field is being read.

---

### Finding Description

`validate_resource_bounds` in `StatefulTransactionValidator` reads the L2 gas price as follows:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

`get_block_info()` is implemented in `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client`, which maps `block_header.l2_gas_price.price_in_fri` into `BlockInfo.gas_prices.strk_gas_prices.l2_gas_price`:

```rust
strk_gas_prices: GasPriceVector {
    ...
    l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
},
``` [2](#0-1) 

The `block_header` also carries a distinct `next_l2_gas_price` field — the EIP-1559 price computed for the **next** block — but `get_block_info_from_sync_client` silently drops it and the `GatewayFixedBlockStateReader` trait returns only `BlockInfo`, which has no slot for `next_l2_gas_price`:

```rust
pub trait GatewayFixedBlockStateReader: Send + Sync {
    async fn get_block_info(&self) -> StarknetResult<BlockInfo>;
    ...
}
``` [3](#0-2) 

`StorageBlockHeader` stores both values as separate fields:

```rust
pub l2_gas_price: GasPricePerToken,
pub next_l2_gas_price: GasPrice,
``` [4](#0-3) 

The threshold check that uses the wrong price is:

```rust
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(...GAS_PRICE_TOO_LOW...);
}
``` [5](#0-4) 

The `min_gas_price_percentage` defaults to `100`, meaning the threshold equals the raw previous-block price: [6](#0-5) 

---

### Impact Explanation

Two divergent failure modes arise whenever `next_l2_gas_price ≠ l2_gas_price`:

**Rising-price market (`next_l2_gas_price > l2_gas_price`):**  
The gateway threshold is lower than the actual execution price. Transactions with `max_price_per_unit` in the range `[threshold × l2_gas_price, threshold × next_l2_gas_price)` pass gateway admission but will be rejected by the blockifier's `check_fee_bounds` at execution time, because the blockifier uses the real next-block price from `ProposalInit`. The gateway incorrectly admits transactions that will fail.

**Falling-price market (`next_l2_gas_price < l2_gas_price`):**  
The gateway threshold is higher than the actual execution price. Transactions with `max_price_per_unit` in the range `[threshold × next_l2_gas_price, threshold × l2_gas_price)` are rejected at the gateway even though they would satisfy the blockifier's fee check. The gateway incorrectly rejects valid transactions.

Both outcomes match the High impact category: **"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

---

### Likelihood Explanation

The EIP-1559 mechanism (`calculate_next_base_gas_price`) adjusts `next_l2_gas_price` every block based on actual gas consumption relative to `gas_target`. Any block that is not exactly at the target produces a `next_l2_gas_price` that differs from `l2_gas_price`: [7](#0-6) 

This is the normal operating condition of the network. The discrepancy grows proportionally to how far utilization deviates from the target, and is bounded only by `gas_price_max_change_denominator`. The bug is therefore triggered continuously during any period of non-uniform block utilization, which is the common case in production.

---

### Recommendation

1. Extend `GatewayFixedBlockStateReader::get_block_info` (or add a new method) to return `next_l2_gas_price` alongside the existing `BlockInfo`.
2. In `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client`, read `block_header.next_l2_gas_price` and surface it.
3. Replace the stale field in `validate_resource_bounds`:

```rust
// Before (wrong):
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;

// After (correct):
let next_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_next_l2_gas_price()   // new method
    .await?;
```

---

### Proof of Concept

1. Observe the last finalized block has `l2_gas_price = 100` and `next_l2_gas_price = 112` (block was over-utilized, EIP-1559 raised the price ~12%).
2. Submit an `InvokeV3` transaction with `l2_gas.max_price_per_unit = 105` and `min_gas_price_percentage = 100`.
3. **Gateway check** (`validate_resource_bounds`): `105 >= 100` → **PASS** (uses stale `l2_gas_price = 100`).
4. Transaction enters the mempool and is included in the next block.
5. **Blockifier check** (`check_fee_bounds`): `105 < 112` → **FAIL** (`MaxGasPriceTooLow`).
6. The transaction reverts at execution; the gateway's admission decision was wrong.

Conversely, with `next_l2_gas_price = 88` (under-utilized block, price fell):
- A transaction with `max_price_per_unit = 92` satisfies `92 >= 88` (correct execution check) but is rejected by the gateway because `92 < 100` (stale threshold). A valid transaction is incorrectly dropped. [8](#0-7) [9](#0-8) [4](#0-3)

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
