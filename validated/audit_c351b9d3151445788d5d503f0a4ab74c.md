### Title
Unprivileged Caller Can Permanently Deny Block Rewards to All Stakers via `update_rewards(disable_rewards: true)` - (File: src/staking/staking.cairo)

### Summary
The `update_rewards` function in the Staking contract carries no access control beyond a basic pause/zero-address check. It accepts a caller-supplied `disable_rewards` boolean. Because `last_reward_block` is written **before** the `disable_rewards` guard is evaluated, any unprivileged address can call `update_rewards(any_valid_staker, true)` once per block to permanently consume the block's reward slot without distributing any yield, denying all stakers their consensus-phase block rewards for that block forever.

### Finding Description
`StakingRewardsManagerImpl::update_rewards` is the sole entry point for distributing per-block consensus rewards to stakers. Its only gate is `general_prerequisites()`, which checks the pause flag and that the caller is non-zero â€” no role restriction exists. [1](#0-0) 

The function immediately writes the current block number to `last_reward_block` **unconditionally**, before inspecting `disable_rewards`: [2](#0-1) 

After that write, if `disable_rewards == true` (or the contract is pre-consensus), the function returns without distributing any rewards: [3](#0-2) 

Any subsequent call in the same block â€” including a legitimate one from the staker â€” is rejected by: [4](#0-3) 

Because `last_reward_block` is a **global** (not per-staker) storage slot, a single attacker call with `disable_rewards: true` exhausts the entire block's reward opportunity for every staker. The rewards that should have been minted and distributed via `_update_rewards` â†’ `reward_supplier_dispatcher.update_unclaimed_rewards_from_staking_contract` are never requested and are permanently lost. [5](#0-4) 

The `disable_rewards` parameter was clearly designed for the consensus layer to mark a block as processed without reward (e.g., a staker that failed to produce a block), but the absence of any caller restriction means any address can weaponise it.

### Impact Explanation
**High â€” Permanent freezing of unclaimed yield.**

An attacker can call `update_rewards(any_active_staker, true)` at the start of every block. Each such call:
1. Marks the block as "already processed" (`last_reward_block = current_block`).
2. Distributes zero rewards to any staker.
3. Cannot be undone â€” the block slot is consumed forever.

Repeated across many blocks, this permanently denies all stakers their consensus-phase block rewards. The yield is never minted into the reward supplier and can never be recovered.

### Likelihood Explanation
**Medium.** The attack requires no special privilege, no capital, and only gas. An attacker can automate one transaction per block. Front-running a legitimate `update_rewards` call is straightforward on Starknet. The only friction is the per-block gas cost, which is negligible compared to the yield denied to stakers.

### Recommendation
Restrict `update_rewards` to a trusted caller (e.g., the consensus/sequencer layer, or a dedicated role). Alternatively, split the function: keep a permissionless `update_rewards(staker_address)` that always distributes rewards, and add a separate role-gated `skip_block_rewards(staker_address)` for the `disable_rewards` path. At minimum, the `last_reward_block` write must not occur before the caller's authorisation to suppress rewards is verified.

### Proof of Concept
1. Consensus rewards are active (`consensus_rewards_first_epoch` is set and current epoch â‰Ą that value).
2. At block `N`, attacker calls `update_rewards(alice_address, disable_rewards: true)` where `alice_address` is any active staker with non-zero balance.
3. `last_reward_block` is written to `N`; no rewards are distributed.
4. Alice (or any other staker) calls `update_rewards(alice_address, false)` in the same block â†’ reverts with `REWARDS_ALREADY_UPDATED`.
5. Block `N`'s reward slot is permanently consumed. All stakers lose their share of block `N`'s rewards.
6. Attacker repeats at block `N+1`, `N+2`, â€¦ indefinitely. [6](#0-5)

        fn update_rewards(
            ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
        ) {
            self.general_prerequisites();
            let current_block_number = starknet::get_block_number();
            assert!(
                current_block_number > self.last_reward_block.read(),
                "{}",
                Error::REWARDS_ALREADY_UPDATED,
            );

            // Assert staker exists and active.
            // Staker is considered to exist from the moment of `stake` (when `InternalStakerInfo`
            // struct is created) until the calling to `unstake_action` (when `InternalStakerInfo`
            // struct is deleted).
            // Staker remains active until the intent period begins, i.e. K epochs after
            // `unstake_intent` is called.
            let staker_info = self.internal_staker_info(:staker_address);
            let curr_epoch = self.get_current_epoch();
            assert!(
                self.is_staker_active(:staker_address, epoch_id: curr_epoch),
                "{}",
                Error::INVALID_STAKER,
            );

            let staker_pool_info = self.staker_pool_info.entry(staker_address).as_non_mut();
            let (staker_total_strk_balance, staker_total_btc_balance) = self
                .get_staker_total_strk_btc_balance_at_epoch(
                    :staker_address, :staker_pool_info, epoch_id: curr_epoch,
                );
            // Assert staker has non-zero balance.
            // Staker exists with zero balance for the first K epochs after `stake`, then the stake
            // becomes effective.
            assert!(staker_total_strk_balance.is_non_zero(), "{}", Error::INVALID_STAKER);

            // Update last block rewards.
            self.last_reward_block.write(current_block_number);

            if disable_rewards || self.is_pre_consensus() {
                return;
            }

            // Get current block data and update rewards.
            let reward_supplier_dispatcher = self.reward_supplier_dispatcher.read();
            let (strk_block_rewards, btc_block_rewards) = self
                .calculate_block_rewards(:reward_supplier_dispatcher, :curr_epoch);
            self
                ._update_rewards(
                    :staker_address,
                    strk_total_rewards: strk_block_rewards,
                    btc_total_rewards: btc_block_rewards,
                    strk_total_stake: staker_total_strk_balance,
                    btc_total_stake: staker_total_btc_balance,
                    :staker_info,
                    :staker_pool_info,
                    :reward_supplier_dispatcher,
                    :curr_epoch,
                );
        }
