### Title
Gateway `skip_stateful_validations` Admits Invoke Transactions With Invalid Signatures Without `__validate__` Check — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `skip_stateful_validations` function in the Apollo gateway's stateful validator skips the `__validate__` entry-point call (which performs signature verification) for invoke transactions with nonce=1 when the sender account has any transaction in the mempool or a recent block. An attacker who first submits a valid `deploy_account` transaction for a fresh address can immediately follow it with an invoke transaction carrying an **invalid or forged signature** at nonce=1. The gateway admits the invalid invoke transaction to the mempool without ever verifying its signature. The transaction is later included in a block and reverts during execution, but the mempool admission invariant — "only cryptographically valid transactions enter the mempool" — is broken.

### Finding Description

`skip_stateful_validations` returns `true` (skip) when three conditions hold simultaneously:

```
tx.nonce() == Nonce(Felt::ONE)
  && account_nonce == Nonce(Felt::ZERO)
  && mempool_client.account_tx_in_pool_or_recent_block(sender).await == true
``` [1](#0-0) 

When `skip_validate = true`, `run_validate_entry_point` constructs `ExecutionFlags { validate: false, … }` and the blockifier's `StatefulValidator::perform_validations` returns immediately without calling `__validate__`: [2](#0-1) [3](#0-2) 

The two earlier checks — `validate_state_preconditions` (resource bounds + nonce range) and `validate_by_mempool` (duplicate/nonce checks in the mempool) — do **not** verify the cryptographic signature: [4](#0-3) 

`account_tx_in_pool_or_recent_block` returns `true` as soon as any transaction for the address is present in the pool: [5](#0-4) 

**Attack path:**

1. Attacker derives a fresh contract address `A` and submits a valid `deploy_account` transaction (nonce=0, correct signature) → gateway admits it to the mempool.
2. Attacker immediately submits an invoke transaction for `A` with nonce=1 and an **arbitrary/invalid signature**.
3. Gateway evaluates: `tx.nonce()==1`, `account_nonce==0`, `account_tx_in_pool_or_recent_block(A)==true` → `skip_validate=true` → `__validate__` is never called → transaction is admitted.
4. Batcher executes: `deploy_account` succeeds (account deployed), invoke transaction runs `__validate__` → invalid signature → transaction **reverts**; nonce is bumped and fees are charged.

The mempool now contains, and the block includes, a transaction whose signature was never verified at admission time.

### Impact Explanation

The gateway's admission invariant — that every accepted transaction has passed its account's `__validate__` entry point — is violated. Invalid (forged-signature) invoke transactions enter the mempool and are sequenced into blocks. While they revert on execution and the attacker pays fees, the sequencer processes and includes cryptographically invalid transactions, wasting block space and execution resources. This matches the allowed High impact: **"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

### Likelihood Explanation

The trigger is straightforward and requires no privileged access: any user can create a fresh account address, submit a valid `deploy_account` transaction, and immediately follow it with an invoke transaction bearing an invalid signature. The `skip_stateful_validations` path is reachable on every gateway instance that has the deploy-account+invoke UX feature enabled (the default). The attacker must pay fees for the reverted invoke, but the admission bypass itself is unconditional once the deploy_account is in the pool.

### Recommendation

Replace the broad `account_tx_in_pool_or_recent_block` check with a check that specifically confirms a **`deploy_account` transaction** (not just any transaction) is present in the mempool for the sender address. Alternatively, instead of skipping `__validate__` entirely, run it against a synthetic "pre-deployed" state that includes the class hash from the pending `deploy_account` transaction, so the signature is still verified before admission.

### Proof of Concept

```
1. Attacker picks salt S, class_hash C → derives address A = calculate_contract_address(S, C, …)
2. POST /gateway/add_transaction  ← valid deploy_account(A, nonce=0, sig=VALID)
   → gateway: account_nonce=0, nonce check passes, __validate_deploy__ runs → ADMITTED
3. POST /gateway/add_transaction  ← invoke(sender=A, nonce=1, sig=0xDEADBEEF…INVALID)
   → gateway: nonce=1, account_nonce=0, account_tx_in_pool=true
   → skip_stateful_validations returns true
   → run_validate_entry_point called with validate=false → __validate__ SKIPPED
   → ADMITTED to mempool
4. Batcher: deploy_account executes (account deployed), invoke executes → __validate__ fails
   → invoke REVERTS, nonce bumped, fee charged
```

The invalid invoke transaction was admitted to the mempool and included in a block without its signature ever being verified by the gateway. [1](#0-0) [6](#0-5) [3](#0-2)

async fn skip_stateful_validations(
    tx: &ExecutableTransaction,
    account_nonce: Nonce,
    mempool_client: SharedMempoolClient,
) -> StatefulTransactionValidatorResult<bool> {
    if let ExecutableTransaction::Invoke(ExecutableInvokeTransaction { tx, .. }) = tx {
        // check if the transaction nonce is 1, meaning it is post deploy_account, and the
        // account nonce is zero, meaning the account was not deployed yet.
        if tx.nonce() == Nonce(Felt::ONE) && account_nonce == Nonce(Felt::ZERO) {
            let account_address = tx.sender_address();
            debug!("Checking if deploy_account transaction exists for account {account_address}.");
            // We verify that a deploy_account transaction exists for this account. It is sufficient
            // to check if the account exists in the mempool since it means that either it has a
            // deploy_account transaction or transactions with future nonces that passed
            // validations.
            return mempool_client
                .account_tx_in_pool_or_recent_block(tx.sender_address())
                .await
                .map_err(|err| mempool_client_err_to_deprecated_gw_err(&tx.signature(), err))
                .inspect(|exists| {
                    if *exists {
                        debug!("Found deploy_account transaction for account {account_address}.");
                    } else {
                        debug!(
                            "No deploy_account transaction found for account {account_address}."
                        );
                    }
                });
        }
    }

    Ok(false)
}
