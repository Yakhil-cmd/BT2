### Title
Gateway Stateful Validator Checks Only L2 Gas Price, Admitting `AllResources` Transactions That Will Always Fail Execution - (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's stateful resource-bounds validator only checks the L2 gas price against the previous block's price. It never checks `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. The mempool's `ValidationArgs` likewise carries only `max_l2_gas_price`. As a result, any `AllResources` V3 transaction with `l1_gas.max_price_per_unit = 0` (or any value below the actual L1 gas price) passes every gateway and mempool admission gate, is stored in the mempool, and then deterministically fails at blockifier pre-validation when `check_fee_bounds` compares all three gas prices against the actual block prices. The nonce is not incremented on that failure, so the attacker can re-submit the same nonce indefinitely.

### Finding Description

**Admission path (no L1/L1-data gas price check):**

`StatefulTransactionValidator::validate_resource_bounds` reads only `strk_gas_prices.l2_gas_price` from the previous block and delegates to `validate_tx_l2_gas_price_within_threshold`, which explicitly handles only the `AllResources` variant and only inspects `l2_gas.max_price_per_unit`. The stateless validator is equally blind to L1 prices — it only enforces `l2_gas.max_price_per_unit >= static_min` and `l2_gas.max_amount <= static_max`: [3](#0-2) 

The `ValidationArgs` struct forwarded to the mempool's `validate_tx` carries only `max_l2_gas_price`; L1 and L1-data gas prices are never transmitted or checked there: [4](#0-3) 

**Execution path (all three gas prices checked):**

`AccountTransaction::check_fee_bounds`, called from `perform_pre_validation_stage` during blockifier execution, iterates over all three resources and rejects the transaction if `resource_bounds.max_price_per_unit < actual_gas_price.get()` for any of them: [5](#0-4) [6](#0-5) 

**The gap:** A transaction with `l1_gas.max_price_per_unit = 0` and a valid `l2_gas.max_price_per_unit` passes every admission check (stateless → stateful → mempool) but will always hit `ResourceBoundsError::MaxGasPriceTooLow { resource: L1Gas, … }` at execution. Because this is a pre-validation failure, the nonce is not incremented, and the transaction can be re-submitted with the same nonce.

### Impact Explanation

An unprivileged attacker can continuously inject transactions that:
1. Pass all gateway and mempool admission checks.
2. Consume mempool capacity (bounded per-account, but the attacker can use many accounts).
3. Force the batcher to dequeue and attempt execution, wasting CPU on `check_fee_bounds` failures.
4. Never advance the account nonce, so the same slot can be re-poisoned immediately after the batcher discards the failed transaction.

This matches **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing**.

### Likelihood Explanation

Trivially reachable: any caller of `starknet_addInvokeTransaction` (or the gateway's `add_tx` endpoint) can set `l1_gas.max_price_per_unit = 0` in the `AllResourceBounds` field. No special privilege, no prior state, and no coordination with other actors is required. The TODO comment confirms the developers are aware the check is incomplete.

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or replace it with a new `validate_all_gas_prices_within_threshold`) to also compare `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` against the corresponding prices from the previous block (applying the same `min_gas_price_percentage` threshold). Propagate these prices through `ValidationArgs` so the mempool can enforce the same check independently. This mirrors the fix applied in the referenced external report: add explicit per-resource lower-bound checks at admission time so that transactions whose bounds are structurally incompatible with current prices are rejected before entering the mempool.

### Proof of Concept

```
// Craft an AllResources invoke V3 transaction with:
//   l1_gas.max_price_per_unit   = 0   (below any real L1 gas price)
//   l1_data_gas.max_price_per_unit = 0
//   l2_gas.max_price_per_unit   = <valid, above threshold>
//   l2_gas.max_amount           = <valid>
//
// 1. POST to gateway /add_transaction
//    → StatelessTransactionValidator::validate_resource_bounds passes
//      (total max_possible_fee > 0 because l2_gas is non-zero; l2_gas price >= min_gas_price)
//    → StatefulTransactionValidator::validate_resource_bounds passes
//      (only l2_gas.max_price_per_unit is compared against previous block price)
//    → mempool.validate_tx passes
//      (ValidationArgs carries only max_l2_gas_price)
//    → Transaction is stored in the mempool.
//
// 2. Batcher calls get_txs, dequeues the transaction, and calls blockifier execute().
//    → perform_pre_validation_stage → check_fee_bounds
//    → l1_gas.max_price_per_unit (0) < actual_l1_gas_price (e.g. 1_000_000_000)
//    → Err(InsufficientResourceBounds { MaxGasPriceTooLow { resource: L1Gas } })
//    → Nonce NOT incremented.
//
// 3. Attacker re-submits the same transaction (same nonce, same invalid bounds).
//    → Cycle repeats indefinitely.
```

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
