### Title
Missing Zero-Address Validation for `reward_address` Allows Permanent Freezing of Staker and Delegator Unclaimed Yield - (File: src/staking/staking.cairo, src/pool/pool.cairo)

---

### Summary
The `stake()` and `change_reward_address()` functions in the staking contract, and `change_reward_address()` in the pool contract, do not validate that the supplied `reward_address` is non-zero. A staker or pool member can register or update their reward address to `0x0`, causing all accrued rewards to be irrecoverably sent to the zero address (or causing a revert that blocks unstaking until the address is corrected).

---

### Finding Description

**Root cause — `stake()` in `src/staking/staking.cairo` (lines 288–366):**

The function validates that `reward_address` is not a registered token address, but performs no zero-address check:

```cairo
assert!(
    !self.does_token_exist(token_address: reward_address),
    "{}",
    GenericError::REWARD_ADDRESS_IS_TOKEN,
);
// ← no assert!(reward_address.is_non_zero(), ...)
```

A staker can call `stake(reward_address: 0x0, operational_address: ..., amount: ...)` and the call succeeds, storing `0x0` as the permanent reward destination.

**Root cause — `change_reward_address()` in `src/staking/staking.cairo` (lines 517–540):**

Same pattern: only the token-address check is present; zero is accepted:

```cairo
assert!(
    !self.does_token_exist(token_address: reward_address),
    "{}",
    GenericError::REWARD_ADDRESS_IS_TOKEN,
);
// ← no zero-address guard
staker_info.reward_address = reward_address;
```

**Root cause — `change_reward_address()` in `src/pool/pool.cairo` (lines 505–526):**

The pool contract mirrors the same omission — only the single-token check is present:

```cairo
assert!(
    self.token_dispatcher.contract_address.read() != reward_address,
    "{}",
    GenericError::REWARD_ADDRESS_IS_TOKEN,
);
// ← no zero-address guard
pool_member_info.reward_address = reward_address;
```

**Contrast with existing practice:** The codebase already defines `GenericError::ZERO_ADDRESS` ("Address is zero") in `src/errors.cairo` (line 20) and enforces a zero-address check on the *caller* via `general_prerequisites()`. The same discipline is simply absent for the `reward_address` parameter.

---

### Impact Explanation

When `reward_address` is `0x0`:

1. **`claim_rewards()`** — rewards are transferred to the zero address. Depending on the ERC-20 implementation, this either silently burns the tokens (permanent loss) or reverts (blocking reward collection until the address is corrected).
2. **`unstake_action()`** — the internal `send_rewards_to_staker()` call attempts to transfer accumulated unclaimed rewards to `0x0` *before* returning the principal. If the ERC-20 reverts on a zero-address transfer, the entire `unstake_action()` call reverts, preventing the staker from ever completing the exit and recovering their staked principal.

Both paths map to **"Permanent freezing of unclaimed yield"** (High) and potentially **"Temporary freezing of funds"** (High) under the allowed impact scope.

---

### Likelihood Explanation

**Low-to-Medium.** The entry path is fully unprivileged — any staker or pool member can trigger it. Realistic scenarios include:

- A front-end or script that passes an uninitialized/default address (`0x0`) for `reward_address`.
- A staker who deliberately sets `reward_address = 0x0` to grief themselves or test the contract.
- A pool member who calls `change_reward_address(0x0)` by mistake.

No privileged role, leaked key, or external dependency is required.

---

### Recommendation

Add an explicit non-zero assertion for `reward_address` in all three entry points:

```cairo
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);
```

Apply this check in:
- `stake()` — `src/staking/staking.cairo` (after line 311)
- `change_reward_address()` — `src/staking/staking.cairo` (after line 523)
- `change_reward_address()` — `src/pool/pool.cairo` (after line 509)

---

### Proof of Concept

```
1. Staker calls:
     staking.stake(reward_address: 0x0, operational_address: <valid>, amount: MIN_STAKE)
   → Succeeds; reward_address stored as 0x0.

2. Staker accrues rewards over several epochs via normal attestation flow.

3. Staker calls:
     staking.unstake_intent()
   → Succeeds; exit window begins.

4. After exit window, staker calls:
     staking.unstake_action(staker_address)
   → Internally calls send_rewards_to_staker(), which attempts:
       token.transfer(recipient: 0x0, amount: unclaimed_rewards)
   → If ERC-20 reverts on zero-address transfer: unstake_action() reverts,
     principal is locked until reward_address is corrected.
   → If ERC-20 silently accepts zero-address transfer: unclaimed rewards
     are permanently burned; principal is returned but yield is lost forever.

        fn unstake_action(ref self: ContractState, staker_address: ContractAddress) -> Amount {
            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self
                .transfer_to_pools_when_unstake(
                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }
