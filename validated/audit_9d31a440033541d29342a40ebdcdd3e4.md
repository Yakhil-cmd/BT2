### Title
Permanent Freezing of Attached NEAR in `fund_xcc_sub_account` When Called Before `factory_update` — (`engine/src/xcc.rs`)

### Summary

When `fund_xcc_sub_account` is called before `factory_update` has ever been invoked (i.e., no XCC router code has been uploaded, so `latest_code_version == CodeVersion::ZERO`), the function incorrectly treats the state as "up to date" and schedules a NEAR transfer to a non-existent sub-account. The transfer promise fails silently, and the caller's attached NEAR is permanently stuck in the engine contract with no refund path.

### Finding Description

`fund_xcc_sub_account` in `engine/src/xcc.rs` accepts an `attached_deposit` (NEAR tokens) and uses `AddressVersionStatus::new(io, latest_code_version, target_code_version)` to decide whether the XCC router sub-account needs to be created or upgraded. [1](#0-0)
