### Title
Blacklisted `adminFeeDestination` Permanently Locks Accumulated Fees and Blocks Fee-Update Admin Operations — (`MetricOmmPool.sol` / `MetricOmmPoolFactory.sol`)

---

### Summary

`collectFees` in `MetricOmmPool.sol` pushes tokens directly to `adminFeeDestination_`. This function is called as a **mandatory prerequisite** inside both `setPoolAdminFees` and `setPoolProtocolFee` in `MetricOmmPoolFactory.sol`. If `adminFeeDestination` is a USDC- or USDT-blacklisted address, every call to `collectFees` reverts, permanently blocking fee-update admin operations and locking all accumulated spread and notional fees inside the pool.

---

### Finding Description

`MetricOmmPool.collectFees` performs direct ERC-20 push transfers to two destinations: [1](#0-0) 

```solidity
if (totalFee0ToAdmin > 0) {
    transferToken0(adminFeeDestination_, totalFee0ToAdmin);   // ← reverts if blacklisted
}
if (totalFee1ToAdmin > 0) {
    transferToken1(adminFeeDestination_, totalFee1ToAdmin);   // ← reverts if blacklisted
}
```

The `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` reset to zero only **after** these transfers succeed: [2](#0-1) 

If the transfer reverts, the entire transaction reverts and the fee accumulators are never cleared — fees remain permanently locked in the pool's accounting.

`collectFees` is not an optional step; it is called unconditionally at the top of both `setPoolAdminFees` and `setPoolProtocolFee` before any state is updated: [3](#0-2) [4](#0-3) 

The publicly callable `collectPoolFees` is also blocked: [5](#0-4) 

`poolAdminFeeDestination` is set at pool creation and can be updated only by the pool admin via `setPoolAdminFeeDestination`, which performs no token transfer and therefore does not revert: [6](#0-5) 

However, until the pool admin acts, the protocol owner (`onlyOwner`) cannot execute `setPoolProtocolFee` for the affected pool, because the call chain always passes through the blocked `collectFees`. The protocol owner has no independent path to change `adminFeeDestination`.

---

### Impact Explanation

1. **Protocol and admin fees permanently locked** — accumulated spread surplus and `notionalFeeToken{0,1}Scaled` cannot be extracted from the pool. This is a direct loss of protocol fees and admin fees.
2. **`setPoolAdminFees` blocked** — the pool admin cannot adjust admin fee rates.
3. **`setPoolProtocolFee` blocked** — the factory owner (protocol) cannot update protocol fee rates for the affected pool, even though they are a higher-privilege actor than the pool admin.
4. **`collectPoolFees` blocked** — any caller attempting fee collection receives a revert.

Core swap and liquidity operations are unaffected, but fee revenue is irrecoverable until the pool admin changes the destination.

---

### Likelihood Explanation

- USDC and USDT (both explicitly in scope) maintain on-chain blacklists. Addresses can be sanctioned post-deployment (e.g., OFAC sanctions, exchange hacks).
- `adminFeeDestination` is set at pool creation and may not be updated for long-lived pools.
- The trigger is accidental (address sanctioned after pool creation) or semi-trusted-admin-induced (pool admin sets a blacklisted address, blocking the protocol owner's fee governance).
- No unprivileged actor is required; the condition arises from normal protocol operation with USDC/USDT as pool tokens.

---

### Recommendation

Replace the push-transfer pattern in `collectFees` with a **pull-based** (claimable balance) model:

1. In `collectFees`, credit `adminFeeDestination` and `FACTORY` in an internal `claimable[token][recipient]` mapping instead of transferring immediately.
2. Add a separate `claimFees(address token)` function that lets each recipient pull their own balance.
3. This way, a blacklisted `adminFeeDestination` only blocks that specific recipient's claim; the protocol's share and all other admin operations remain unaffected.

---

### Proof of Concept

**Attack / accidental flow:**

1. Pool is created with `token0 = USDC`, `adminFeeDestination = 0xABCD`.
2. Swaps accumulate spread surplus and notional fees over time.
3. USDC blacklists `0xABCD` (e.g., OFAC sanction).
4. Anyone calls `collectPoolFees(pool)` → `pool.collectFees(...)` → `USDC.safeTransfer(0xABCD, ...)` → **reverts**.
5. Pool admin calls `setPoolAdminFees(pool, newFee, newFee)` → same revert path → **blocked**.
6. Factory owner calls `setPoolProtocolFee(pool, newFee, newFee)` → same revert path → **blocked**.
7. `notionalFeeToken0Scaled` and `notionalFeeToken1Scaled` are never zeroed; all accumulated fees remain locked in the pool indefinitely.
8. Recovery requires the pool admin to call `setPoolAdminFeeDestination(pool, newDest)` — but if the pool admin is unresponsive or the admin key is compromised, the protocol owner has no independent remediation path. [7](#0-6) [8](#0-7)

  function collectFees(
    uint256 protocolSpreadFeeE6_,
    uint256 adminSpreadFeeE6_,
    uint256 protocolNotionalFeeE8_,
    uint256 adminNotionalFeeE8_,
    address adminFeeDestination_
  ) external onlyFactory nonReentrant(PoolActions.COLLECT_FEES) {
    uint256 spreadSumE6;
    uint256 notionalSumE8;
    unchecked {
      spreadSumE6 = protocolSpreadFeeE6_ + adminSpreadFeeE6_;
      notionalSumE8 = protocolNotionalFeeE8_ + adminNotionalFeeE8_;
      if (spreadSumE6 == 0 && notionalSumE8 == 0) {
        return;
      }
    }

    uint256 notionalFee0AmountScaled = notionalFeeToken0Scaled;
    uint256 notionalFee1AmountScaled = notionalFeeToken1Scaled;

    uint256 surplus0Scaled =
      balance0() * TOKEN_0_SCALE_MULTIPLIER - uint256(binTotals.scaledToken0) - notionalFee0AmountScaled;
    uint256 surplus1Scaled =
      balance1() * TOKEN_1_SCALE_MULTIPLIER - uint256(binTotals.scaledToken1) - notionalFee1AmountScaled;

    unchecked {
      uint256 spreadFee0ToAdminScaled = spreadSumE6 == 0 ? 0 : (surplus0Scaled * adminSpreadFeeE6_) / spreadSumE6;
      uint256 spreadFee1ToAdminScaled = spreadSumE6 == 0 ? 0 : (surplus1Scaled * adminSpreadFeeE6_) / spreadSumE6;

      uint256 spreadFee0ToProtocolScaled = spreadSumE6 == 0 ? 0 : (surplus0Scaled * protocolSpreadFeeE6_) / spreadSumE6;
      uint256 spreadFee1ToProtocolScaled = spreadSumE6 == 0 ? 0 : (surplus1Scaled * protocolSpreadFeeE6_) / spreadSumE6;

      uint256 notionalFee0ToAdminScaled =
        notionalSumE8 == 0 ? 0 : (notionalFee0AmountScaled * adminNotionalFeeE8_) / notionalSumE8;
      uint256 notionalFee1ToAdminScaled =
        notionalSumE8 == 0 ? 0 : (notionalFee1AmountScaled * adminNotionalFeeE8_) / notionalSumE8;

      uint256 notionalFee0ToProtocolScaled = notionalFee0AmountScaled - notionalFee0ToAdminScaled;
      uint256 notionalFee1ToProtocolScaled = notionalFee1AmountScaled - notionalFee1ToAdminScaled;

      uint256 totalFee0ToAdminScaled = spreadFee0ToAdminScaled + notionalFee0ToAdminScaled;
      uint256 totalFee1ToAdminScaled = spreadFee1ToAdminScaled + notionalFee1ToAdminScaled;

      uint256 totalFee0ToProtocolScaled = spreadFee0ToProtocolScaled + notionalFee0ToProtocolScaled;
      uint256 totalFee1ToProtocolScaled = spreadFee1ToProtocolScaled + notionalFee1ToProtocolScaled;

      (uint256 totalFee0ToAdmin, uint256 totalFee1ToAdmin) =
        deltasScaledToExternal(totalFee0ToAdminScaled, totalFee1ToAdminScaled, Math.Rounding.Floor);
      (uint256 totalFee0ToProtocol, uint256 totalFee1ToProtocol) =
        deltasScaledToExternal(totalFee0ToProtocolScaled, totalFee1ToProtocolScaled, Math.Rounding.Floor);

      if (totalFee0ToAdmin > 0) {
        transferToken0(adminFeeDestination_, totalFee0ToAdmin);
      }
      if (totalFee1ToAdmin > 0) {
        transferToken1(adminFeeDestination_, totalFee1ToAdmin);
      }
      if (totalFee0ToProtocol > 0) {
        transferToken0(FACTORY, totalFee0ToProtocol);
      }
      if (totalFee1ToProtocol > 0) {
        transferToken1(FACTORY, totalFee1ToProtocol);
      }

      notionalFeeToken0Scaled = 0;
      notionalFeeToken1Scaled = 0;

      emit ProtocolFeesCollected(totalFee0ToProtocol, totalFee1ToProtocol, totalFee0ToAdmin, totalFee1ToAdmin);
    }
  }
