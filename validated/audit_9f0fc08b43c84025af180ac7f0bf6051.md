### Title
Missing Zero-Address Validation on `reward_address` Causes Temporary Freezing of Staker Funds - (`src/staking/staking.cairo`, `src/pool/pool.cairo`)

### Summary

Both `stake()` and `change_reward_address()` in `staking.cairo`, and `enter_delegation_pool()` and `change_reward_address()` in `pool.cairo`, accept a `reward_address` parameter with no zero-address guard. The stored `reward_address` is later used as the recipient of a `checked_transfer` call. If `reward_address` is zero, the STRK token (OpenZeppelin ERC20) reverts on any transfer to `address(0)`, causing `claim_rewards()` and `unstake_action()` to revert unconditionally, temporarily freezing unclaimed yield and the staker's principal.

### Finding Description

**Root cause — `stake()` in `staking.cairo`:** [1](#0-0) 

The function validates that `reward_address` is not a registered token address, but performs **no zero-address check**. A caller may pass `reward_address = 0` and the record is written to storage at line 338.

**Root cause — `change_reward_address()` in `staking.cairo`:** [2](#0-1) 

Same pattern: only the token-address guard is present; zero is accepted and written to storage at line 530.

**Root cause — `enter_delegation_pool()` and `change_reward_address()` in `pool.cairo`:** [3](#0-2) [4](#0-3) 

Both accept `reward_address` with only a token-address guard; zero is accepted.

**Sink — `send_rewards_to_staker()` in `staking.cairo`:** [5](#0-4) 

`checked_transfer` is called unconditionally with `recipient: reward_address`. OpenZeppelin ERC20 reverts on any transfer to `address(0)`, including zero-amount transfers, because the zero-address guard precedes the amount check.

**`unstake_action()` calls `send_rewards_to_staker` unconditionally:** [6](#0-5) 

If `reward_address = 0`, line 495 (`send_rewards_to_staker`) reverts before the principal transfer at line 506 and before pool cleanup at lines 509–513. The staker's principal and all delegated pool balances remain locked in the staking contract.

**Sink — `claim_rewards()` in `pool.cairo`:** [7](#0-6) 

Line 366 transfers to `reward_address`; if zero, every `claim_rewards` call for that pool member reverts, freezing their unclaimed yield.

### Impact Explanation

- **Temporary freezing of staker principal**: A staker with `reward_address = 0` cannot execute `unstake_action()` — the call reverts at `send_rewards_to_staker` before any funds move. The staker's STRK principal remains locked in the staking contract until they call `change_reward_address()` to set a valid address.
- **Temporary freezing of unclaimed yield**: Both `claim_rewards()` in `staking.cairo` (via `send_rewards_to_staker`) and `claim_rewards()` in `pool.cairo` revert, making accumulated rewards inaccessible.

The freeze is technically self-recoverable (the staker/member can call `change_reward_address()` first), but the window between setting `reward_address = 0` and recovery constitutes a **temporary freezing of funds**, which is within the allowed impact scope.

### Likelihood Explanation

- Any unprivileged staker or pool member can trigger this by passing `reward_address = 0` to `stake()` or `enter_delegation_pool()`, or by calling `change_reward_address(address(0))`.
- Accidental triggering is realistic: a misconfigured script, a frontend bug, or a zero-initialized variable can produce a zero address.
- No privileged access, no external dependency compromise, and no bridge interaction is required — the entry path is a direct public call.

### Recommendation

Add a zero-address guard in all four entry points:

```cairo
// staking.cairo — stake() and change_reward_address()
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);

// pool.cairo — enter_delegation_pool() and change_reward_address()
assert!(reward_address.is_non_zero(), "{}", GenericError::ZERO_ADDRESS);
```

Optionally, add a guard inside `send_rewards_to_staker` as a defense-in-depth measure:

```cairo
if amount.is_non_zero() {
    assert!(reward_address.is_non_zero(), "reward address is zero");
    token_dispatcher.checked_transfer(recipient: reward_address, amount: amount.into());
}
```

### Proof of Concept

1. Attacker (or accidental user) calls `stake(reward_address: 0, operational_address: X, amount: MIN_STAKE)`.
2. Rewards accumulate in `staker_info.unclaimed_rewards_own` over time.
3. Staker calls `unstake_intent()` — succeeds (no transfer involved).
4. After the exit wait window, staker calls `unstake_action(staker_address)`.
5. Inside `unstake_action`, `send_rewards_to_staker` is called unconditionally at line 495.
6. `send_rewards_to_staker` calls `token_dispatcher.checked_transfer(recipient: 0, amount: rewards)`.
7. The STRK ERC20 contract reverts: transfer to `address(0)` is forbidden.
8. `unstake_action` reverts; the staker's principal remains locked in the staking contract.
9. The staker must call `change_reward_address(valid_address)` before `unstake_action` can succeed.

The same flow applies to pool members: `enter_delegation_pool(reward_address: 0, amount: X)` followed by `claim_rewards(pool_member)` reverts at line 366 of `pool.cairo`, freezing all accumulated pool rewards.

            // Prerequisites and asserts.
            self.general_prerequisites();
            let mut staker_info = self.internal_staker_info(:staker_address);
            let unstake_time = staker_info
                .unstake_time
                .expect_with_err(Error::MISSING_UNSTAKE_INTENT);
            assert!(Time::now() >= unstake_time, "{}", GenericError::INTENT_WINDOW_NOT_FINISHED);

            // Send rewards to staker's reward address.
            // It must be part of this function's flow because staker_info is about to be erased.
            let token_dispatcher = strk_token_dispatcher();
            self.send_rewards_to_staker(:staker_address, ref :staker_info, :token_dispatcher);
            // Update staker info to storage (it will be erased later).
            // This is done here to avoid re-entrancy.
            self.write_staker_info(:staker_address, :staker_info);

            let staker_amount = self.get_own_balance(:staker_address).to_strk_native_amount();
            let staker_pool_info = self.staker_pool_info.entry(staker_address);
            self.remove_staker(:staker_address, :staker_info, :staker_pool_info);

            // Return stake to staker.
            token_dispatcher
                .checked_transfer(recipient: staker_address, amount: staker_amount.into());
            // Return delegated stake to pools and zero their balances.
            self

                    :staker_address, staker_pool_info: staker_pool_info.as_non_mut(),
                );
            // Clear staker pools.
            staker_pool_info.pools.clear();
            staker_amount
        }

        fn claim_rewards(ref self: ContractState, pool_member: ContractAddress) -> Amount {
            // Asserts.
            let mut pool_member_info = self.internal_pool_member_info(:pool_member);
            let caller_address = get_caller_address();
            let reward_address = pool_member_info.reward_address;
            assert!(
                caller_address == pool_member || caller_address == reward_address,
                "{}",
                Error::POOL_CLAIM_REWARDS_FROM_UNAUTHORIZED_ADDRESS,
            );

            let until_checkpoint = self.get_current_checkpoint(:pool_member);

            // Calculate rewards and update entry_to_claim_from.
            let (mut rewards, updated_entry_to_claim_from) = self
                .calculate_rewards(
                    :pool_member,
                    from_checkpoint: pool_member_info.reward_checkpoint,
                    :until_checkpoint,
                    entry_to_claim_from: pool_member_info.entry_to_claim_from,
                );
            rewards += pool_member_info._unclaimed_rewards_from_v0;
            pool_member_info._unclaimed_rewards_from_v0 = Zero::zero();
            pool_member_info.entry_to_claim_from = updated_entry_to_claim_from;
            pool_member_info.reward_checkpoint = until_checkpoint;

            // Write the updated pool member info to storage.
            self.write_pool_member_info(:pool_member, :pool_member_info);

            // Transfer rewards to the pool member.
            let reward_token = IERC20Dispatcher { contract_address: STRK_TOKEN_ADDRESS };
            reward_token.checked_transfer(recipient: reward_address, amount: rewards.into());

            // Emit event.
            self
                .emit(
                    Events::PoolMemberRewardClaimed {
                        pool_member, reward_address, amount: rewards,
                    },
                );

            rewards
        }
