### Title
Missing Signer Authorization in `RegisterEncryptionKey` Allows Any User to Overwrite Any Address's Encryption Key — (File: `x/e2ee/keeper/keeper.go`)

### Summary
The `x/e2ee` module's `RegisterEncryptionKey` message handler stores an encryption key for an arbitrary `Address` field supplied in the message without verifying that the transaction signer owns that address. Any unprivileged user can overwrite any validator's registered public key, corrupting the e2ee key state and preventing affected validators from decrypting the admin-issued blocklist.

### Finding Description
`MsgRegisterEncryptionKey` carries two fields: `Address` (the account whose key is being registered) and `Key` (the age public key to store). The keeper's handler unconditionally writes the supplied key for the supplied address:

```go
// x/e2ee/keeper/keeper.go  lines 45-53
func (k Keeper) RegisterEncryptionKey(
    ctx context.Context,
    req *types.MsgRegisterEncryptionKey,
) (*types.MsgRegisterEncryptionKeyResponse, error) {
    if err := k.registerEncryptionKey(ctx, req.Address, []byte(req.Key)); err != nil {
        return nil, err
    }
    return &types.MsgRegisterEncryptionKeyResponse{}, nil
}
```

There is no check that the Cosmos SDK transaction signer equals `req.Address`. The Cosmos SDK authenticates *who signed the transaction* but the keeper never compares that signer to the address being written. Any account can therefore register (or overwrite) the encryption key for any other account, including validators. [1](#0-0) 

The downstream consumer of these keys is the `ProposalHandler.SetBlockList` in `app/proposal.go`, which decrypts the admin-issued blocklist blob using the validator's private `age.Identity`:

```go
// app/proposal.go  lines 226-228
reader, err := age.Decrypt(bytes.NewBuffer(blob), h.Identity)
if err != nil {
    return err
}
```

The admin encrypts the blocklist against the public keys stored in the e2ee registry. If an attacker has replaced a validator's registered key with an attacker-controlled key, the admin's next encrypted blob will be encrypted to the wrong public key, and the targeted validator's `age.Decrypt` call will fail with `ErrIncorrectIdentity`. [2](#0-1) 

### Impact Explanation
- **e2ee key/message state corruption with direct security impact (High):** The attacker permanently overwrites a validator's registered public key. Every subsequent blocklist update encrypted by the admin will be unreadable by the targeted validator.
- **Blocklist authorization bypass (High):** A validator that cannot decrypt the blocklist operates with an empty or stale blocklist (`SetBlockList` returns an error and the map is not updated). Addresses that the admin intended to block can have their transactions included in blocks proposed or processed by the affected validator, bypassing the block-list enforcement mechanism. [3](#0-2) 

### Likelihood Explanation
The attack requires only a standard signed Cosmos transaction with a crafted `MsgRegisterEncryptionKey`. No special permissions, leaked keys, or privileged access are needed. The message is a normal on-chain transaction reachable by any account with enough gas.

### Recommendation
In `RegisterEncryptionKey`, verify that the transaction signer matches `req.Address` before writing:

```go
func (k Keeper) RegisterEncryptionKey(
    ctx context.Context,
    req *types.MsgRegisterEncryptionKey,
) (*types.MsgRegisterEncryptionKeyResponse, error) {
    sdkCtx := sdk.UnwrapSDKContext(ctx)
    signers := sdkCtx.TxSigner() // or extract from msg server context
    // Ensure req.Address == signer
    signerAddr, err := k.addressCodec.StringToBytes(req.Address)
    if err != nil {
        return nil, err
    }
    if !bytes.Equal(signers, signerAddr) {
        return nil, sdkerrors.ErrUnauthorized.Wrap("signer does not match address")
    }
    if err := k.registerEncryptionKey(ctx, req.Address, []byte(req.Key)); err != nil {
        return nil, err
    }
    return &types.MsgRegisterEncryptionKeyResponse{}, nil
}
```

Alternatively, ignore `req.Address` entirely and derive the address from the verified transaction signer.

### Proof of Concept
1. Validator V has registered their legitimate age public key `pubkey_V` for address `addr_V`.
2. Attacker A submits `MsgRegisterEncryptionKey{Address: "addr_V", Key: "age1<attacker_pubkey>"}` signed by attacker's own account.
3. The keeper writes `attacker_pubkey` to the store under `addr_V` with no error.
4. Admin encrypts the next blocklist blob for all validators using their registered keys; the blob for `addr_V` is now encrypted to `attacker_pubkey`.
5. Validator V calls `SetBlockList(blob)` → `age.Decrypt` returns `ErrIncorrectIdentity` → `SetBlockList` returns an error → validator's in-memory blocklist is not updated.
6. Validator V proposes and processes blocks without applying the current blocklist; blocked addresses can transact freely through V. [4](#0-3) [5](#0-4)

// SetBlockList don't fail if the identity is not set or the block list is empty.
func (h *ProposalHandler) SetBlockList(blob []byte) error {
	if h.Identity == nil {
		return nil
	}

	if bytes.Equal(h.lastBlockList, blob) {
		return nil
	}
	h.lastBlockList = make([]byte, len(blob))
	copy(h.lastBlockList, blob)

	if len(blob) == 0 {
		h.blocklist = make(map[string]struct{})
		return nil
	}

	reader, err := age.Decrypt(bytes.NewBuffer(blob), h.Identity)
	if err != nil {
		return err
	}

	data, err := io.ReadAll(reader)
	if err != nil {
		return err
	}

	var blocklist BlockList
	if err := json.Unmarshal(data, &blocklist); err != nil {
		return err
	}

	// convert to map
	m := make(map[string]struct{}, len(blocklist.Addresses))
	for _, s := range blocklist.Addresses {
		addr, err := h.addressCodec.StringToBytes(s)
		if err != nil {
			return fmt.Errorf("invalid bech32 address: %s, err: %w", s, err)
		}
		if IsUnblockable(addr) {
			continue
		}
		encoded, err := h.addressCodec.BytesToString(addr)
		if err != nil {
			return fmt.Errorf("invalid bech32 address: %s, err: %w", s, err)
		}
		m[encoded] = struct{}{}
	}

	h.blocklist = m
	return nil
}
