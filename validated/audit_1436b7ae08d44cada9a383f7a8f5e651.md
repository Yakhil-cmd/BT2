### Title
Unrestricted `update_rewards` Caller with `disable_rewards` Flag Allows Any Address to Permanently Freeze Staker Consensus Block Rewards — (File: `src/staking/staking.cairo`)

---

### Summary

`IStakingRewardsManager::update_rewards` has no caller restriction. Any unprivileged address can invoke it with `disable_rewards: true`, which advances `last_reward_block` to the current block without distributing any rewards. Because the contract enforces a strict one-call-per-block invariant, the legitimate block proposer is then locked out for that block. Repeated every block, this permanently freezes all consensus-era staker yield.

---

### Finding Description

`update_rewards` is the entry point for distributing per-block consensus rewards to a staker. Its only access guard is `general_prerequisites()`, which checks that the contract is not paused and that the caller is not the zero address: [1](#0-0) 

```cairo
fn update_rewards(
    ref self: ContractState, staker_address: ContractAddress, disable_rewards: bool,
) {
    self.general_prerequisites();
    let current_block_number = starknet::get_block_number();
```

There is no assertion that the caller is the block proposer, the attestation contract, or any other privileged address. Compare this with `update_rewards_from_attestation_contract`, which correctly gates access: [2](#0-1) 

```cairo
fn update_rewards_from_attestation_contract(...) {
    self.general_prerequisites();
    assert!(self.is_pre_consensus(), ...);
    self.assert_caller_is_attestation_contract();
```

No equivalent `assert_caller_is_*` guard exists in `update_rewards`.

The one-call-per-block invariant is enforced immediately after the prerequisites check: [3](#0-2) 

```cairo
assert!(
    current_block_number > self.last_reward_block.read(),
    "{}",
    Error::REWARDS_ALREADY_UPDATED,
);
```

`last_reward_block` is then unconditionally written to storage **before** the `disable_rewards` branch is evaluated: [4](#0-3) 

```cairo
self.last_reward_block.write(current_block_number);

if disable_rewards || self.is_pre_consensus() {
    return;
}
```

An attacker who calls `update_rewards(victim_staker, disable_rewards: true)` in any block will:
1. Pass all checks (not paused, caller non-zero, staker active, staker has non-zero balance).
2. Advance `last_reward_block` to the current block.
3. Return early, distributing zero rewards.
4. Prevent any subsequent call in the same block from succeeding (the `REWARDS_ALREADY_UPDATED` assertion will revert it).

---

### Impact Explanation

This is a **High** impact finding: **Permanent freezing of unclaimed yield**.

In the consensus-rewards era (`is_pre_consensus() == false`), all staker STRK block rewards flow exclusively through `update_rewards`. If an attacker front-runs every block's legitimate call with `disable_rewards: true`, the staker accrues zero rewards indefinitely. The staker's principal is unaffected, but all future yield is permanently frozen. The attack is sustainable at the cost of gas per block.

---

### Likelihood Explanation

**High.** The function is public, requires no special role, and the `disable_rewards` flag is a plain boolean parameter. Any address with enough gas to submit one transaction per block can execute this attack against any staker. There is no economic barrier and no on-chain mechanism to detect or prevent it.

---

### Recommendation

Restrict `update_rewards` to a single authorized caller — the designated block proposer contract or a role-gated address — analogously to how `update_rewards_from_attestation_contract` restricts its caller via `assert_caller_is_attestation_contract`. At minimum, remove the `disable_rewards` parameter from the public interface and handle that logic internally based on on-chain state (e.g., whether the staker attested), so no external caller can suppress reward distribution.

---

### Proof of Concept

```
// Attacker script (pseudocode, executed every block)
for each new block B:
    staking_contract.update_rewards(
        staker_address = VICTIM,
        disable_rewards = true   // any caller can set this
    )
    // last_reward_block is now B; no rewards distributed
    // legitimate proposer's call in block B will revert with REWARDS_ALREADY_UPDATED
```

Concrete call path:
1. Attacker (any EOA) calls `Staking::update_rewards(victim, true)`.
2. `general_prerequisites()` passes — contract not paused, caller non-zero. [5](#0-4) 
3. Block-number guard passes (first call this block). [3](#0-2) 
4. Staker-active checks pass for any live staker. [6](#0-5) 
5. `last_reward_block` written to current block. [7](#0-6) 
6. `disable_rewards == true` → early return, zero rewards distributed. [8](#0-7) 
7. Any subsequent legitimate call in the same block reverts at step 3.
