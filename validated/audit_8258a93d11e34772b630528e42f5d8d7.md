### Title
Incorrect Multi-NFT Royalty Calculation in WalletConnect Offer Confirmation Understates User Spending - (File: `packages/gui/src/electron/commands/parseCommandDisplay.ts`)

### Summary
The `formatAmountWithRoyalties` function in `parseCommandDisplay.ts` incorrectly divides the base amount by the number of NFTs before computing each royalty, causing the WalletConnect `chia_wallet.take_offer` confirmation dialog to display a systematically underestimated "You Spend" total whenever an offer involves two or more NFTs with royalties. A user who approves based on the displayed figure actually spends more XCH or CAT than they consented to.

### Finding Description
`formatAmountWithRoyalties` is called by `walletDeltaToDisplay` → `parseCommandDisplay` to produce the `amountWithRoyalties` string shown in the WalletConnect confirmation dialog for `chia_wallet.take_offer`. [1](#0-0) 

The faulty line is:

```typescript
const splitAmount = amount / BigInt(royaltyPercentages.length);   // line 363
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

This computes `(amount / N) * Σ(royaltyPct_i) / 10_000` instead of the correct `Σ(amount * royaltyPct_i / 10_000)`. The two expressions are equal only when `N = 1`. For `N ≥ 2` the displayed royalty total is exactly `1/N` of the correct value.

**Concrete example from the project's own test** (`parseCommandDisplay.test.ts` lines 251–334):

| Parameter | Value |
|---|---|
| Amount | 100,000,000 mojos (0.0001 XCH) |
| NFT 1 royalty | 500 bps (5%) |
| NFT 2 royalty | 10 bps (0.1%) |
| **Displayed** `amountWithRoyalties` | **0.00010255 XCH** |
| **Correct** `amountWithRoyalties` | **0.0001051 XCH** |

The test asserts `0.00010255`, confirming the bug is baked in and passing. [2](#0-1) 

The `amountWithRoyalties` field is rendered directly in the WalletConnect confirmation dialog's "You Spend" section: [3](#0-2) 

### Impact Explanation
When a WalletConnect-connected dApp sends `chia_wallet.take_offer` with an offer containing ≥ 2 NFTs that carry royalties, the confirmation dialog shows a "You Spend" total that is lower than the amount the blockchain will actually deduct. The underestimation factor is `1/N` applied to the royalty portion. With high royalty percentages and many NFTs the gap is substantial (e.g., 10 NFTs each at 10% royalty: displayed total = 1.1× base, actual total = 2× base). The user approves a transaction believing they are spending less than they actually will, constituting approval of the wrong amount — a direct wallet security impact.

### Likelihood Explanation
Any WalletConnect-connected dApp can craft a `chia_wallet.take_offer` payload containing multiple NFTs with royalties. No special privileges are required beyond an established WalletConnect session, which is a normal user-initiated action. The bug is triggered deterministically whenever `royaltyPercentages.length > 1`.

### Recommendation
Replace the `splitAmount` division with a per-NFT calculation over the full `amount`:

```typescript
const royaltyAmount = royaltyPercentages.reduce(
  (total, royaltyPercentage) => total + (amount * BigInt(royaltyPercentage)) / 10_000n,
  0n,
);
```

Remove the `splitAmount` variable entirely. Update the corresponding test assertion in `parseCommandDisplay.test.ts` to expect `0.0001051` instead of `0.00010255`.

### Proof of Concept
1. Establish a WalletConnect session with the Chia GUI wallet.
2. Send `chia_wallet.take_offer` with an offer string encoding:
   - **Offered**: 2 NFTs, each with a 5% royalty (500 bps).
   - **Requested**: 1,000,000,000,000 mojos (1 XCH).
3. Observe the confirmation dialog:
   - **Displayed** "You Spend": `1.05 XCH` (royalties shown as 5%, i.e., `1/2` of correct).
   - **Actual** deduction enforced by blockchain: `1.10 XCH` (5% + 5% = 10% royalties).
4. User approves at 1.05 XCH; wallet spends 1.10 XCH — 0.05 XCH more than consented to.

The root cause is in `formatAmountWithRoyalties` at line 363 of `packages/gui/src/electron/commands/parseCommandDisplay.ts`. [4](#0-3)

```typescript
function formatAmountWithRoyalties(
  line: DisplayWalletDeltaItem,
  amount: bigint,
  royaltyPercentages: number[],
): string | undefined {
  if (royaltyPercentages.length === 0 || line.kind === 'nft') {
    return undefined;
  }

  const splitAmount = amount / BigInt(royaltyPercentages.length);
  const royaltyAmount = royaltyPercentages.reduce(
    (total, royaltyPercentage) => total + (splitAmount * BigInt(royaltyPercentage)) / 10_000n,
    0n,
  );
  const totalAmount = amount + royaltyAmount;

  if (line.kind === 'xch') {
    return mojoToChiaLocaleString(totalAmount);
  }

  return mojoToCATLocaleString(totalAmount);
}
```
