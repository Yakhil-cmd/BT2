### Title
Missing Lower-Bound Enforcement on Threshold in FROST Signing Allows Sub-Threshold Signing Sessions - (File: src/frost/mod.rs)

### Summary
`assert_sign_inputs` in `src/frost/mod.rs` validates that `threshold ≤ participants.len()` but never checks `threshold ≥ 2`. Every other validation gate in the library (`assert_key_invariants` in `src/dkg.rs`, `validate_triple_inputs` in the OT-ECDSA path) enforces the lower bound. A malicious coordinator can invoke `sign_v1`, `sign_v2` (EdDSA), or `sign` (RedJubjub) with `threshold = 1`, constructing a `KeyPackage` with `min_signers = 1` and bypassing the threshold security property established during DKG.

### Finding Description
`assert_sign_inputs` is the shared validation gate for all FROST-based signing (EdDSA and RedJubjub). It performs two threshold checks:

1. Upper bound: `threshold.value() > participants.len()` → `ThresholdTooLarge`
2. **No lower bound check** — `threshold < 2` is never tested. [1](#0-0) 

By contrast, `assert_key_invariants` in `src/dkg.rs` explicitly enforces `threshold ≥ 2`: [2](#0-1) 

And `validate_triple_inputs` in the OT-ECDSA triple generation path also enforces `threshold ≥ 2`: [3](#0-2) 

The `ReconstructionLowerBound` type itself imposes no minimum — it is a plain `usize` wrapper with no invariant: [4](#0-3) 

When `threshold = 1` is accepted by `assert_sign_inputs`, it is forwarded directly into `construct_key_package`, which passes it as `min_signers` to the upstream FROST `KeyPackage::new`: [5](#0-4) 

The same pattern exists in the RedJubjub path: [6](#0-5) 

With `min_signers = 1`, the FROST library's `aggregate()` call accepts a `signature_shares` map containing only a single entry. The coordinator, who controls the aggregation step, can therefore attempt aggregation with only their own share, regardless of how many participants contributed.

### Impact Explanation
A malicious coordinator who calls `sign_v1`/`sign_v2`/`sign` with `threshold = 1`:

- Causes `KeyPackage::new` to be constructed with `min_signers = 1` for every honest participant.
- The FROST `aggregate()` function's share-count guard (`signature_shares.len() < min_signers`) is trivially satisfied by a single share.
- If the key material was generated with `threshold = 1` (e.g., via an external dealer or the test-utility `build_key_packages_with_dealer` with `min_signers = 1`), each participant's key share equals the full secret key (degree-0 polynomial). The coordinator alone can produce a cryptographically valid threshold signature without any other participant's contribution — an unauthorized creation of a valid threshold signature.
- Even when the DKG was run with `threshold ≥ 2`, honest participants complete their protocol steps (committing nonces, sending signature shares) and receive a corrupted or failed signing output. Their nonces are consumed and cannot be safely reused (FROST nonce-reuse is a key-recovery vulnerability), permanently invalidating that signing session.

This maps to: **High — Corruption of sign outputs so honest parties accept unusable cryptographic outputs**, and in the dealer-key-generation scenario, **Critical — Unauthorized creation of a valid threshold signature**.

### Likelihood Explanation
The entry point is the public `sign_v1` / `sign_v2` / `sign` API. Any caller who controls the `threshold` argument (a plain `usize` or `ReconstructionLowerBound`) can trigger this. The coordinator role is the natural attacker position. The inconsistency is a single missing guard that exists in every other validation function in the codebase, making accidental omission the most likely cause and exploitation straightforward.

### Recommendation
Add the same lower-bound guard that exists in `assert_key_invariants` and `validate_triple_inputs` to `assert_sign_inputs`:

```rust
if threshold.value() < 2 {
    return Err(InitializationError::ThresholdTooSmall {
        threshold: threshold.value(),
        min: 2,
    });
}
```

This should be inserted immediately after the `participants.len() < 2` check in `assert_sign_inputs` (`src/frost/mod.rs`, line 127). The same guard should be added to the OT-based ECDSA `sign` function in `src/ecdsa/ot_based_ecdsa/sign.rs` (line 57 area), which also lacks a lower-bound check on `threshold`.

pub fn assert_sign_inputs(
    participants: &[Participant],
    threshold: impl Into<ReconstructionLowerBound>,
    me: Participant,
    coordinator: Participant,
) -> Result<ParticipantList, InitializationError> {
    let threshold = threshold.into();
    if participants.len() < 2 {
        return Err(InitializationError::NotEnoughParticipants {
            participants: participants.len(),
        });
    }
    let Some(participants) = ParticipantList::new(participants) else {
        return Err(InitializationError::DuplicateParticipants);
    };

    // ensure my presence in the participant list
    if !participants.contains(me) {
        return Err(InitializationError::MissingParticipant {
            role: "self",
            participant: me,
        });
    }

    // validate threshold
    if threshold.value() > participants.len() {
        return Err(InitializationError::ThresholdTooLarge {
            threshold: threshold.value(),
            max: participants.len(),
        });
    }

    // ensure the coordinator is a participant
    if !participants.contains(coordinator) {
        return Err(InitializationError::MissingParticipant {
            role: "coordinator",
            participant: coordinator,
        });
    }
    Ok(participants)
}

fn construct_key_package(
    threshold: ReconstructionLowerBound,
    me: Participant,
    keygen_output: &KeygenOutput,
) -> Result<KeyPackage, ProtocolError> {
    let identifier = me.to_identifier()?;
    let signing_share = keygen_output.private_share;
    let verifying_share = signing_share.into();
    let verifying_key = keygen_output.public_key;
    let key_package = KeyPackage::new(
        identifier,
        signing_share,
        verifying_share,
        verifying_key,
        u16::try_from(threshold.value()).map_err(|_| {
            ProtocolError::Other("threshold cannot be converted to u16".to_string())
        })?,
    );

    // Ensures the values are zeroized on drop
    Ok(key_package)
}
